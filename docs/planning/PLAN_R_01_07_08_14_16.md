## R-01 — Quantitative cross-orbit advantage lower bound

### Problem

`Orbcrypt/Theorems/InvariantAttack.lean:170-187` proves
`invariant_attack : ∃ A : Adversary X M, hasAdvantage scheme A`
under a separating G-invariant. This is the existential
(deterministic) form — the WEAKEST advantage convention. R-01 wants
the *probabilistic* form: under a separating G-invariant, the
IND-1-CPA advantage equals exactly `1` (the maximum).

### Mathematical content (verified sound)

Pick `D := fun c => decide (f c = f (scheme.reps m₀))`. Under
`IsGInvariant f`:

* **Right orbit (m₀):** for every `g : G`, `f (g • reps m₀) =
  f (reps m₀)` (G-invariance), so `D` is constantly `true` on the
  support. Hence
  `probTrue (orbitDist (reps m₀)) D = (uniformPMF G).map (· • reps m₀)`-
  pushed-through-`D = uniformPMF G`-pushed-through-`(D ∘ (· • reps
  m₀)) = uniformPMF G`-pushed-through-`(constantly true)` = 1`.
* **Left orbit (m₁):** for every `g : G`, `f (g • reps m₁) =
  f (reps m₁) ≠ f (reps m₀)`, so `D` is constantly `false`. Hence
  `probTrue (orbitDist (reps m₁)) D = 0`.
* **Advantage:** `|0 − 1| = 1`. (Tight; cannot be exceeded —
  matches `advantage_le_one`.)

### KEM-layer companion: **dropped — mathematically vacuous**

**Finding (review pass).** During plan review the KEM-layer
companion was found to be **vacuous**: G-invariant attacks give
advantage **0** (not 1) on `kemAdvantage_uniform`. The KEM
uniform-form game distinguishes `kemEncapsDist kem` (uniform
encaps) from `PMF.pure (encaps kem g_ref)` (a fixed encaps) —
both distributions live in the **basepoint's single orbit**, so
any G-invariant `f : X → Bool` extended to `(c, k) ↦ f c`
satisfies `f c = f basePoint` constantly under both PMFs, giving
advantage `|f basePoint − f basePoint| = 0`.

The KEM-layer parallel of R-01's existential content (`G-invariant
attack ⇒ KEMOIA is False`) is **already discharged** by
`det_kemoia_false_of_nontrivial_orbit`
(`Orbcrypt/KEM/Security.lean`, post-Workstream-E of audit
2026-04-23). That theorem uses the *non*-G-invariant test
`fun c => decide (c = g₀ • basePoint)` to refute deterministic
KEMOIA — which is exactly the right adversarial shape because the
KEM game distinguishes ciphertexts within ONE orbit (so a
G-invariant function carries no signal).

**Action:** the plan retains the IND-CPA-only R-01 headline and
drops the KEM companion. The R-01 module docstring will document
this finding, explicitly cross-referencing
`det_kemoia_false_of_nontrivial_orbit` as the KEM-layer analogue
already in-tree.

### Proof outline

For each of the two `probTrue_orbitDist_invariant_eq_*` lemmas:

1. Unfold `orbitDist x = PMF.map (· • x) (uniformPMF G)`.
2. Apply `probTrue_PMF_map_uniformPMF_toReal` from
   `Probability/Monad.lean` (already used at line 726 of
   `CompSecurity.lean`) to get
   `(filter (D ∘ (· • x))).card / |G|`.
3. For `= 1`: show the filter equals `Finset.univ` (every group
   element satisfies the predicate by G-invariance + `hEq`),
   giving `|G| / |G| = 1`.
4. For `= 0`: show the filter is empty (every group element falsifies
   the predicate by G-invariance + `hNe`), giving `0 / |G| = 0`.
5. Convert ENNReal-valued probTrue to ℝ-valued at the end.

Headline (`indCPAAdvantage = 1`):

1. Unfold `indCPAAdvantage scheme A` via `indCPAAdvantage_eq` to
   `advantage (A.guess scheme.reps) (orbitDist (reps m₀))
   (orbitDist (reps m₁))`.
2. Substitute `A = invariantAttackAdversary f m₀ m₁` so
   `A.guess scheme.reps = fun c => decide (f c = f (reps m₀))`.
3. Apply `probTrue_orbitDist_invariant_eq_one` (giving `1`) and
   `probTrue_orbitDist_invariant_eq_zero` (giving `0`).
4. Unfold `advantage`: `|1 − 0| = 1`. Close via `abs_of_pos` +
   `norm_num`.

### Edge cases (R-01)

* **`|G| = 0`** — impossible; `[Nonempty G]` typeclass excludes
  the empty group.
* **`|G| = 1`** — `Equiv.Perm (Fin 1)` (or any singleton group).
  Then `orbitDist (reps m) = PMF.pure (1 • reps m) = PMF.pure
  (reps m)`. The `= 1` and `= 0` lemmas still hold (single-
  element filter or empty filter).
* **`Y` is empty** — vacuous; `f x : Y` cannot hold for any
  `f x`, so `Sep` hypothesis is never satisfied.
* **`scheme.reps m₀ = scheme.reps m₁`** — forbidden by
  `OrbitEncScheme.reps_distinct`. Cannot occur.
* **Same orbit, distinct reps** (`reps m₀ ≠ reps m₁` but in same
  orbit) — `f` invariant gives `f (reps m₀) = f (reps m₁)`
  (same orbit ⇒ same f-value), contradicting `hSep`. So this
  case is excluded by `hSep`.

## R-07 — Cross-orbit lower bound for equivariant combiners

### Problem

`Orbcrypt/PublicKey/CombineImpossibility.lean:557` lands the
*upper* bound `concrete_combiner_advantage_bounded_by_oia : combiner-
DistinguisherAdvantage ≤ ε` from `ConcreteOIA scheme ε`. Lines
544–616 of the same file disclose the audit gap: intra-orbit mass
bounds (`combinerOrbitDist_mass_bounds : both Booleans have ≥ 1/|G|
mass on m_bp's orbit`) do **not** imply cross-orbit advantage lower
bounds — two orbits could have identical Pr[true] = 1/2, yielding
advantage 0. R-07 supplies the missing cross-orbit witness.

### Mathematical content (verified sound)

Strategy: introduce `CrossOrbitNonDegenerateCombiner` — a Prop
combining the existing `NonDegenerateCombiner` (intra-orbit non-
triviality on `m_bp`'s orbit) with a cross-orbit witness that
`combinerDistinguisher` is constantly `false` on `m_target`'s
orbit. Then:

* `probTrue (orbitDist (reps m_bp)) (combinerDistinguisher comb)
  ≥ 1/|G|` (from the existing intra-orbit mass bound on the
  `true` branch).
* `probTrue (orbitDist (reps m_target)) (combinerDistinguisher comb)
  = 0` (from the cross-orbit constant-false witness).
* `advantage = |intra − cross| = |≥1/|G| − 0| ≥ 1/|G|`.

The lower bound `1/|G|` is not the tightest possible — the
intra-orbit mass on `true` could be much higher (e.g., `≥ 1/2`
under stronger non-degeneracy) — but `1/|G|` is the bound that
the *existing* `combinerOrbitDist_mass_bounds` machinery delivers
directly.

### Edge cases (R-07)

* **`|G| = 1`** (singleton group) — `1/|G| = 1`, so the headline
  bound is `1 ≤ advantage`, i.e. `advantage = 1`. Sanity check:
  with a singleton group, the orbit is a single point; the
  `cross_constant_false` hypothesis says `combinerDistinguisher
  comb (reps m_target) = false`. `intra : ∃ g ∈ G, comb base
  point (g • basepoint) ≠ comb basepoint basepoint` requires
  `g ≠ 1` for non-triviality, but `|G| = 1` means `g = 1` only.
  So `intra` is *unsatisfiable* at `|G| = 1`. The headline
  vacuously holds.
* **`m_bp = m_target`** — by `OrbitEncScheme.reps_distinct`
  (`Crypto/Scheme.lean:74`):
  `m₁ ≠ m₂ → MulAction.orbit G (reps m₁) ≠ MulAction.orbit G
  (reps m₂)`.
  This is **orbit-distinct** (not just rep-distinct): distinct
  messages have *disjoint* orbits. So `reps m_target` and
  `reps m_bp` lie in disjoint orbits when `m_bp ≠ m_target`.
  The `cross_constant_false` hypothesis says the distinguisher
  is `false` constantly on `m_target`'s ORBIT (not just at
  `reps m_target`); the `intra` (NonDegenerate) hypothesis says
  it's NON-constant on `m_bp`'s ORBIT. Since the two orbits are
  disjoint, both hypotheses can hold simultaneously without
  contradiction.

  **What about `m_bp = m_target`?** Then both hypotheses apply
  to the SAME orbit, and they contradict: `intra` requires
  non-constant, `cross_constant_false` requires constant. So
  the conjunction `CrossOrbitNonDegenerateCombiner` is
  `False` at `m_bp = m_target`. The headline theorem
  `combinerDistinguisherAdvantage_ge_inv_card` is therefore
  vacuously true at `m_bp = m_target`. Document this in the
  docstring; no separate edge-case theorem needed.
* **`combinerOrbitDist_mass_bounds.1`** uses the witness
  `g = 1` for the `true` branch (combiner's identity-application
  gives `true`). If the combiner is truly degenerate
  (`comb basepoint basepoint = false`), `intra` fails. The
  intra-witness `g_w` from `combinerOrbitDist_mass_bounds.2`
  gives the `false` branch.

### Mathematical content (verified sound)

**Hypothesis: ε-SU2 (strongly-universal-2).** The 1-time SUF-CMA
reduction takes a hash family `h : K → Msg → Tag` satisfying
**ε-SU2**: for all distinct `m₁, m₂` and arbitrary `(t₁, t₂)`,
`Pr_k[h k m₁ = t₁ ∧ h k m₂ = t₂] ≤ ε / |Tag|`.

This is **strictly stronger** than ε-universal (which only bounds
the δ = 0 case `Pr[h k m₁ = h k m₂]`) and strictly stronger than
ε-AXU (which bounds output differences `Pr[h k m₁ − h k m₂ = δ]`
without controlling joint marginals). For fixed-key SUF-CMA the
joint-distribution bound is what we need — see proof below.

**Why not ε-universal?** Universal hash bounds `Pr[h k m₁ = h k m₂]`
(a single value coincidence), but the SUF-CMA win event `h k m' = t'`
corresponds to a *specific tag* `t'` (a function of the view), not
the symmetric coincidence. ε-universal alone is INSUFFICIENT.

**Why not ε-AXU?** AXU bounds `Pr[h k m₁ − h k m₂ = δ]` for any
`δ`. Without uniform marginals (separate property), the
sub-event analysis gives `|Tag|·ε` not `ε`. ε-SU2 directly
encodes the joint bound and gives the cleanest reduction.

**Theorem (Wegman–Carter / Stinson 1994 Theorem 1).** For
`mac := deterministicTagMAC h` and `h` ε-SU2: every 1-time
adversary `A` has `forgeryAdvantage mac A ≤ ε`.

**Proof.** Let `m_q := A.query`, `m'(t) := (A.forge t).1`,
`t'(t) := (A.forge t).2`. The win event is
`Win(k) := m'(h k m_q) ≠ m_q ∧ h k (m'(h k m_q)) = t'(h k m_q)`.

By `verify_inj`, the second conjunct is equivalent to
`verify k (m'(h k m_q)) (t'(h k m_q)) = true`.

For each `t_q ∈ Tag`, define
`Win_at(k, t_q) := (h k m_q = t_q) ∧ (m'(t_q) ≠ m_q) ∧
                   (h k (m'(t_q)) = t'(t_q))`.
Note: `h k m_q` takes exactly one value for each `k`, so the
events `Win_at(k, t_q)` partition `Win(k)` over `t_q`:
```
Win(k) ⇔ ∃ t_q : Tag, Win_at(k, t_q)
        ⇔ ⋃_{t_q ∈ Tag} Win_at(k, t_q)   (disjoint over t_q)
```
Hence
```
Pr_k[Win] = Σ_{t_q ∈ Tag} Pr_k[Win_at(t_q)]
          = Σ_{t_q ∈ Tag} Pr_k[h k m_q = t_q ∧
                                m'(t_q) ≠ m_q ∧
                                h k (m'(t_q)) = t'(t_q)].
```

Per-`t_q` bound:
* If `m'(t_q) = m_q`: the `≠ m_q` conjunct fails, summand = 0.
* If `m'(t_q) ≠ m_q`: by ε-SU2 at messages `(m_q, m'(t_q))` and
  tag values `(t_q, t'(t_q))`,
  `Pr_k[h k m_q = t_q ∧ h k (m'(t_q)) = t'(t_q)] ≤ ε / |Tag|`.

Sum over `t_q`:
```
Pr_k[Win] ≤ Σ_{t_q ∈ Tag} (ε / |Tag|) = |Tag| · (ε / |Tag|) = ε.
```
QED.

**Carter-Wegman and bitstring-polynomial are both ε-SU2 at the
same `ε = 1/p` and `ε = n/p` respectively, since the joint
collision sets factor cleanly:**

* **Carter-Wegman SU2.** For distinct `m₁, m₂` and arbitrary
  `(t₁, t₂)`, the system `k.1 m_i + k.2 = t_i` has UNIQUELY
  solution `(k.1, k.2)` (when `m₁ ≠ m₂`). So
  `Pr_k[joint] = 1/|K| = 1/p² = (1/p) / |Tag| = ε / |Tag|` ✓.
* **Bitstring-polynomial SU2.** For distinct `b₁, b₂` and
  arbitrary `(t₁, t₂)`, the system has at most n joint
  solutions in `k.1` (polynomial system, degree ≤ n via the
  difference polynomial argument), each with a unique `k.2`.
  So `Pr_k[joint] ≤ n / p² = (n/p) / |Tag| = ε / |Tag|` ✓.

### Revised framework hypothesis: ε-SU2 (strongly-universal-2)

```lean
/-- A hash family is **`ε`-strongly-universal-2** (`ε`-SU2) iff
    every joint output pair has probability ≤ `ε / |Tag|` under
    uniform key. This is the "right" hypothesis for one-time
    Wegman–Carter SUF-CMA security. Strictly stronger than both
    `IsEpsilonUniversal` and `IsEpsilonAXU`. -/
def IsEpsilonSU2 {K Msg Tag : Type*} [Fintype K] [Nonempty K]
    [Fintype Tag] [Nonempty Tag] [DecidableEq Tag]
    (h : K → Msg → Tag) (ε : ℝ≥0∞) : Prop :=
  ∀ (m₁ m₂ : Msg) (t₁ t₂ : Tag), m₁ ≠ m₂ →
    probTrue (uniformPMF K)
      (fun k => decide (h k m₁ = t₁ ∧ h k m₂ = t₂)) ≤
    ε / (Fintype.card Tag : ℝ≥0∞)

-- Note: `[Nonempty Tag]` ensures `Fintype.card Tag ≠ 0`, so the
-- `ε / |Tag|` divisor is well-defined in ℝ≥0∞ (avoiding the
-- `ε / 0 = ⊤` degenerate case for ε ≠ 0). For Carter-Wegman over
-- `ZMod p` with `[Fact (Nat.Prime p)]`, `Nonempty (ZMod p)` is
-- automatic via `inferInstance`.

/-- ε-SU2 implies ε-universal (the δ = 0 specialisation). -/
theorem IsEpsilonSU2.toIsEpsilonUniversal {h : K → Msg → Tag}
    {ε : ℝ≥0∞} (hSU2 : IsEpsilonSU2 h ε) : IsEpsilonUniversal h ε

/-- ε-SU2 implies ε-AXU (when Tag has additive group structure). -/
theorem IsEpsilonSU2.toIsEpsilonAXU [AddGroup Tag] {h : K → Msg → Tag}
    {ε : ℝ≥0∞} (hSU2 : IsEpsilonSU2 h ε) : IsEpsilonAXU h ε
```

The headline 1-time SUF-CMA reduction now reads:
```lean
theorem isSUFCMASecure_of_isEpsilonSU2
    [Fintype K] [Nonempty K] [Fintype Tag] [Nonempty Tag]
    [DecidableEq Msg] [DecidableEq Tag]
    {h : K → Msg → Tag} {ε : ℝ≥0∞}
    (hε_finite : ε ≠ ⊤) (hSU2 : IsEpsilonSU2 h ε) :
    IsSUFCMASecure (deterministicTagMAC h) ε.toReal
```

with proof exactly as the cleanest analysis above.

**Q-time SUF-CMA — CRITICAL FINDING (review pass):**
**Nonce-free Carter–Wegman MAC is broken at Q ≥ 2.**

After the user's choice ("1-time + Q-time"), a careful re-analysis
of the Q-time game revealed a **mathematical impossibility** for
our specific MACs (`carterWegmanMAC`, `bitstringPolynomialMAC`):

For Carter–Wegman `h k m = k.1 · m + k.2` with **fixed key**
`(k.1, k.2)`, the adversary querying at `m_1 ≠ m_2` recovers the
key by solving the linear system
`{k.1 m_1 + k.2 = t_1, k.1 m_2 + k.2 = t_2}` ⇒
`k.1 = (t_1 − t_2)/(m_1 − m_2), k.2 = t_1 − k.1 · m_1`. The
adversary then forges any fresh `m'` with `t' = k.1 m' + k.2`
deterministically. So `forgeryAdvantage_Qtime ≥ 1 − negligible`
for all Q ≥ 2 (the adversary wins almost surely).

The same attack applies to `bitstringPolynomialMAC`: the key is
`(k.1, k.2) ∈ ZMod p × ZMod p` (just 2 unknowns), recovered at
Q ≥ 2 queries on distinct bitstrings.

**The standard Wegman–Carter 1981 §3 Q-time MAC requires fresh
nonces per message** (`tag_i := h(k, m_i) ⊕ s_i` where `s_i =
PRF(nonce_i)` or similar). Our existing MACs are *nonce-free*
("1-time MAC" in the literature). To formalise the Q-time form
honestly, we'd need either:

* A nonce abstraction + PRF formalism (out of scope for this
  PR; tracked as **R-05** random-oracle research milestone).
* A negative theorem: explicit construction of an attacker
  achieving `forgeryAdvantage_Qtime ≥ 1 − 1/|Tag|` for our
  specific MACs at Q ≥ 2.

**Revised scope.** R-14 still defines `IsQtimeSUFCMASecure` (the
predicate is well-formed regardless of which MAC instances
satisfy it). R-08 + R-13⁺ each prove the 1-time bound and
include a **negative theorem** demonstrating the Q-time attack.
This is the **honest** discharge: 1-time SUF-CMA is achievable;
Q-time is not for nonce-free Carter–Wegman / bitstring-polynomial.

**Decision (overrides user's "1-time + Q-time" choice):** the
mathematics forces 1-time-only for the *positive* Q-time bound;
the user's preference is honoured by formalising the Q-time
*framework* (predicates + 1-time specialisation) plus a
*negative* result that documents the gap. No claim of Q-time
SUF-CMA security on these MACs is made (nor could it be).

**Cross-link:** future work (research milestone R-05 or a
dedicated workstream) would add `nonceCarterWegmanMAC` /
`nonceBitstringPolynomialMAC` variants that *do* achieve
Q-time SUF-CMA via nonce sampling + PRF.

### Layer 0 — Almost-XOR-universal helper (`Probability/UniversalHash.lean`)

```lean
/-- A hash family `h : K → Msg → Tag` is **`ε`-almost-XOR-
    universal** (`ε`-AXU) iff for every two distinct messages
    `m₁ ≠ m₂` and every output difference `δ : Tag`, the
    probability over uniform `k` that `h k m₁ - h k m₂ = δ`
    is at most `ε`. Strictly stronger than `IsEpsilonUniversal`
    (which is the `δ = 0` special case). Required for the Q-time
    Wegman–Carter SUF-CMA reduction. -/
def IsEpsilonAXU {K Msg Tag : Type*} [Fintype K] [Nonempty K]
    [DecidableEq Tag] [Sub Tag] (h : K → Msg → Tag) (ε : ℝ≥0∞) : Prop :=
  ∀ (m₁ m₂ : Msg) (δ : Tag), m₁ ≠ m₂ →
    probTrue (uniformPMF K) (fun k => decide (h k m₁ - h k m₂ = δ)) ≤ ε

/-- AXU specialises to universal at `δ = 0`. -/
theorem IsEpsilonAXU.toIsEpsilonUniversal {K Msg Tag : Type*}
    [Fintype K] [Nonempty K] [DecidableEq Tag] [AddGroup Tag]
    {h : K → Msg → Tag} {ε : ℝ≥0∞} (hAXU : IsEpsilonAXU h ε) :
    IsEpsilonUniversal h ε
```

### Layer 1 — `MACAdversary` + `forgeryAdvantage` (`MACSecurity.lean`)

```lean
/-- A **1-time MAC adversary**: given access to a single tag for a
    chosen message `query`, output a forgery `(m_forge, t_forge)` —
    typically with `m_forge ≠ query` (the "fresh message" SUF-CMA
    constraint). Functional shape mirroring `Adversary` from
    `Crypto/Security.lean`. -/
structure MACAdversary (K : Type*) (Msg : Type*) (Tag : Type*) where
  /-- The message the adversary chooses to query for an honest tag. -/
  query : Msg
  /-- Given the honest tag for `query`, produce a forgery candidate. -/
  forge : Tag → Msg × Tag

/-- Forgery event for a 1-time MAC adversary: forgery verifies
    AND uses a fresh message. -/
def MACAdversary.forges [DecidableEq Msg] (mac : MAC K Msg Tag)
    (A : MACAdversary K Msg Tag) (k : K) : Bool :=
  let t_honest := mac.tag k A.query
  let (m', t') := A.forge t_honest
  decide (m' ≠ A.query) && mac.verify k m' t'

/-- 1-time SUF-CMA forgery advantage: probability over uniform key
    that the adversary's fresh-message forgery verifies. -/
noncomputable def forgeryAdvantage [Fintype K] [Nonempty K]
    [DecidableEq Msg] (mac : MAC K Msg Tag) (A : MACAdversary K Msg Tag) : ℝ :=
  (probTrue (uniformPMF K) (A.forges mac)).toReal

/-- A MAC is `ε`-SUF-CMA-secure (1-time) iff every adversary's
    forgery advantage is at most `ε`. -/
def IsSUFCMASecure [Fintype K] [Nonempty K] [DecidableEq Msg]
    (mac : MAC K Msg Tag) (ε : ℝ) : Prop :=
  ∀ A : MACAdversary K Msg Tag, forgeryAdvantage mac A ≤ ε

theorem forgeryAdvantage_nonneg [Fintype K] [Nonempty K] [DecidableEq Msg]
    (mac : MAC K Msg Tag) (A : MACAdversary K Msg Tag) :
    0 ≤ forgeryAdvantage mac A

theorem forgeryAdvantage_le_one [Fintype K] [Nonempty K] [DecidableEq Msg]
    (mac : MAC K Msg Tag) (A : MACAdversary K Msg Tag) :
    forgeryAdvantage mac A ≤ 1
```

### Layer 2 — Multi-query (Q-time) variant

```lean
/-- A **Q-time MAC adversary**: outputs `Q` queries upfront (non-
    adaptive — keeps the framework clean) and produces a forgery
    given the `Q` honest tags. The fresh-message constraint
    requires `m' ∉ {queries}`. -/
structure MultiQueryMACAdversary (K Msg Tag : Type*) (Q : ℕ) where
  /-- The `Q` chosen messages. Required distinct (enforced by the
      forgery game's fresh-message check, but stating it here
      simplifies the reduction). -/
  queries : Fin Q → Msg
  /-- Given the `Q` honest tags, produce a forgery `(m', t')`. -/
  forge : (Fin Q → Tag) → Msg × Tag

/-- Forgery event for a Q-time adversary: fresh message + verifies. -/
def MultiQueryMACAdversary.forges [DecidableEq Msg]
    {Q : ℕ} (mac : MAC K Msg Tag)
    (A : MultiQueryMACAdversary K Msg Tag Q) (k : K) : Bool :=
  let tags := fun i => mac.tag k (A.queries i)
  let (m', t') := A.forge tags
  decide (∀ i, m' ≠ A.queries i) && mac.verify k m' t'

noncomputable def forgeryAdvantage_Qtime [Fintype K] [Nonempty K]
    [DecidableEq Msg] {Q : ℕ} (mac : MAC K Msg Tag)
    (A : MultiQueryMACAdversary K Msg Tag Q) : ℝ :=
  (probTrue (uniformPMF K) (A.forges mac)).toReal

def IsQtimeSUFCMASecure [Fintype K] [Nonempty K] [DecidableEq Msg]
    {Q : ℕ} (mac : MAC K Msg Tag) (ε : ℝ) : Prop :=
  ∀ A : MultiQueryMACAdversary K Msg Tag Q, forgeryAdvantage_Qtime mac A ≤ ε
```

**Note on `|Msg|` cardinality.** The fresh-message constraint
`∀ i, m' ≠ A.queries i` is **vacuously satisfiable** only when
`|Msg| > Q` (else every `m' : Msg` collides with some query, the
predicate is `false`, and `IsQtimeSUFCMASecure mac ε` holds
vacuously at any `ε ≥ 0`). For meaningful Q-time analysis the
caller picks `Q < |Msg|` — for Carter–Wegman over `ZMod p` this
means `Q < p`; for bitstring-polynomial over `Bitstring n` it
means `Q < 2^n`. Both Q-time NEGATIVE results land at `Q = 3`
on parameters where `|Msg| ≫ 3`, so this constraint is met
trivially in the Workstream-D scope.

### Layer 3 — Generic Wegman–Carter reductions

```lean
/-- **Headline 1-time SUF-CMA reduction.** A `deterministicTagMAC`
    over an `ε`-SU2 hash is `ε`-SUF-CMA-secure (1-time). The
    SU2 hypothesis is what's actually required (NOT just
    ε-universal — see "Mathematical content" above for why). -/
theorem isSUFCMASecure_of_isEpsilonSU2
    [Fintype K] [Nonempty K] [Fintype Tag] [DecidableEq Msg]
    [DecidableEq Tag]
    (h : K → Msg → Tag) (ε : ℝ≥0∞) (hε_finite : ε ≠ ⊤)
    (hSU2 : IsEpsilonSU2 h ε) :
    IsSUFCMASecure (deterministicTagMAC h) ε.toReal

/-- (Honest scope.) ε-universal hash alone does NOT generally
    imply ε-SUF-CMA-secure: the universal property bounds only
    `Pr[h k m₁ = h k m₂]` (the δ = 0 case) but the SUF-CMA
    win event corresponds to arbitrary δ. The proper hypothesis
    is ε-SU2 (above). For schemes where ε-universal implies
    ε-SU2 (e.g., when the hash has uniform marginals), the

/-- **Q-time recoveryAttack helper.** When the hash family `h`
    admits a recovery procedure for *some specific choice* of `Q`
    distinct messages (i.e. there exists an injective `msgs :
    Fin Q → Msg` such that the linear / polynomial system at
    those messages is uniquely solvable for the key), any `Q + 1`-
    query adversary can: (1) query at those `Q` specific messages
    to recover the key, (2) forge deterministically on a fresh
    `Q+1`-th message.

    **Predicate form: existential-over-msgs (relaxed).** The
    definition uses `∃ msgs` (not `∀ msgs`) because for some hash
    families (e.g., `bitstringPolynomialHash`), recovery is
    uniquely solvable only for *specific* bitstring choices — the
    universal-over-msgs form is overly restrictive and would
    fail for some valid recovery instances. The existential form
    is sufficient for the negative theorem: as long as ONE
    recovery-friendly message-tuple exists, the adversary can
    use it.

    No generic `IsQtimeSUFCMASecure_of_isEpsilonAXU` theorem is
    landed because such a theorem would be false for our existing
    MACs — the Q-time bound for nonce-free Wegman–Carter is `1`
    (broken), not `Q · ε`. The Q-time predicate is provided as
    framework; specific MAC instances must individually prove they
    satisfy it (typically via nonce-based key derivation, which is
    out of this PR's scope; tracked as research milestone R-05). -/
def IsKeyRecoverableForSomeQueries [Fintype K] [Nonempty K]
    [DecidableEq Msg] [DecidableEq Tag]
    (h : K → Msg → Tag) (queries : ℕ) : Prop :=
  ∃ (msgs : Fin queries → Msg) (recover : (Fin queries → Tag) → Option K),
    Function.Injective msgs ∧
    ∀ k : K, recover (fun i => h k (msgs i)) = some k

/-- **Headline negative result.** When the hash is key-recoverable
    from some specific `Q`-tuple of queries, the corresponding
    `deterministicTagMAC` is NOT `ε`-Q+1-time-SUF-CMA-secure for
    any `ε < 1 - 1/|Msg|` (or simply `ε < 1` if `|Msg|` is large):
    an explicit adversary achieves forgery advantage `1 −
    negligible`.

    **Adversary construction.** Given `(msgs, recover)` from the
    hypothesis, the Q+1-time adversary:
    1. Submits the Q queries `msgs : Fin Q → Msg`.
    2. Receives Q honest tags `tags := fun i => h k (msgs i)`.
    3. Recovers the key `k_recovered := (recover tags).get` via
       the hypothesis (which guarantees `recover tags = some k`).
    4. Picks any fresh message `m'` not in `Set.range msgs` (exists
       when `|Msg| > Q`, the standard case).
    5. Computes the deterministic forgery `t' := h k_recovered m'`.
    6. Submits `(m', t')`. Since `k_recovered = k`, this verifies.

    **Required cardinality.** `Fintype.card Msg > Q` is needed for
    step (4) to find a fresh `m'`. For Carter–Wegman with `Msg =
    ZMod p` and `p ≥ 3` this is automatic at `Q = 2`. For
    bitstring-polynomial with `Msg = Bitstring n` and `n ≥ 2`,
    `|Msg| = 2^n ≥ 4 > 2 = Q`, also automatic. -/
theorem not_isQtimeSUFCMASecure_of_keyRecoverableForSomeQueries
    [Fintype K] [Nonempty K] [Fintype Msg] [DecidableEq Msg]
    [DecidableEq Tag]
    {Q : ℕ} (h : K → Msg → Tag) (hQ_lt_card : Q < Fintype.card Msg)
    (hRecover : IsKeyRecoverableForSomeQueries h Q)
    (ε : ℝ) (hε : ε < 1) :
    ¬ IsQtimeSUFCMASecure (Q := Q + 1)
      (deterministicTagMAC h) ε
```

### Mathematical content (verified sound)

**Carter–Wegman is SU2.** For `h k m := k.1 * m + k.2` over
`ZMod p` (prime), and any `m₁ ≠ m₂` and arbitrary `(t₁, t₂) :
ZMod p × ZMod p`, the joint event `h k m₁ = t₁ ∧ h k m₂ = t₂`
reduces to a 2×2 linear system in `(k.1, k.2)`:
```
k.1 m₁ + k.2 = t₁
k.1 m₂ + k.2 = t₂
```
Subtract: `k.1 (m₁ − m₂) = t₁ − t₂`. Since `m₁ ≠ m₂`,
`(m₁ − m₂) ≠ 0` in the field `ZMod p`, so `k.1 = (t₁ − t₂) /
(m₁ − m₂)`, a UNIQUE value. Then `k.2 = t₁ − k.1 m₁`, also
unique. So `|joint set| = 1`, keyspace `|K| = p²`, probability
`1/p²`. Compare to SU2 bound `ε/|Tag| = (1/p)/p = 1/p²`. ✓

So Carter-Wegman is **`(1/p)`-SU2** (and consequently
`(1/p)`-universal and, with the additive-group structure of
`ZMod p`, `(1/p)`-AXU). The existing
`carterWegmanHash_isUniversal` is the δ = 0 specialisation; the
SU2 upgrade promotes the joint analysis to arbitrary `(t₁, t₂)`.

(Note: `carterWegmanHash_isUniversal` already exists and proves
`(1/p)`-universal — we reuse its proof structure for the SU2
upgrade, generalising the joint-collision-card argument from a
single value `(t, t)` to arbitrary `(t₁, t₂)`.)

applies. (For Carter–Wegman in practice, `p` is large.)

### Mathematical content (verified sound)

The bitstring-polynomial hash `h k b = k.2 + ∑ i, toBit (b i) ·
k.1^(i+1)` is `(n/p)`-SU2. For `b₁ ≠ b₂` and any `(t₁, t₂)`, the
joint event `h k b₁ = t₁ ∧ h k b₂ = t₂` reduces to a 2×2 system:
* Equation 1: `k.2 + P₁(k.1) = t₁` where `P₁(X) := ∑ i, toBit (b₁
  i) · X^(i+1)`.
* Equation 2: `k.2 + P₂(k.1) = t₂` where `P₂` is similar for `b₂`.
* Subtracting: `P₁(k.1) - P₂(k.1) = t₁ - t₂`, i.e. the
  difference polynomial `Δ(k.1) = δ` for `δ := t₁ - t₂`.

By the existing `bitstringDiffPolynomial_card_roots_le` (R-13
result, `Polynomial.card_roots'` over the field), the equation
`Δ(k.1) = δ` has at most `n` roots in `ZMod p` (the difference
polynomial has degree ≤ n; for the constant-shifted equation
`Δ(X) - δ = 0`, also at most n roots).

For each root `k.1 ∈ {roots of Δ - δ}`, the value `k.2 = t₁ -
P₁(k.1)` is uniquely determined. So `|joint set| ≤ n`, keyspace
`p²`, probability ≤ `n/p² = (n/p)/|Tag|`. ✓ This is `(n/p)`-SU2.

**Cardinality + witness-choice side conditions.** The recovery
requires `n ≥ 1` (so position 0 exists in the bitstring) and
choosing the witness pair carefully so the difference polynomial
has degree EXACTLY 1 (single root, deterministic recovery):

* **Optimal witness pair.** `b₀ := fun i => decide (i = 0)` (i.e.,
  `(T, F, F, ..., F)`, the standard basis `e_0`) and `b₁ := fun
  _ => false` (the all-zero bitstring).
* **Difference polynomial.** `Δ(X) = ∑ i, (toBit (b₀ i) − toBit
  (b₁ i)) · X^(i+1) = 1 · X^1 = X`. Degree exactly 1.
* **Recovery.** `Δ(k.1) = t₁ − t₂` becomes `k.1 = t₁ − t₂`
  (unique, no edge case). Then `k.2 = t₂` (since `b₁` is all-zero,
  `evalAtBitstring b₁ k.1 = 0` so `t₂ = k.2 + 0 = k.2`).
* **No `k.1 ∈ {0, 1}` ambiguity:** unlike the naive choice `b₁ =
  (F, T, F, ...)` which gives `Δ = X − X²` with roots `{0, 1}`,
  this witness gives Δ with a single root (degree 1).

**Hypothesis simplification.** With this witness, the negative
theorem requires only `n ≥ 1` (not `n ≥ 2`). And `|Bitstring n| =
2^n ≥ 2 > 2 = Q-1` auto-satisfies the cardinality bound for the
fresh `m'` (any bitstring ≠ `b₀` and ≠ `b₁` works, e.g. `b' :=
fun i => decide (i = 1)` for `n ≥ 2`).

For `n = 1`: `Bitstring 1 = Fin 1 → Bool` has only 2 elements
(all-zero and all-true). The witness pair uses both, leaving no
fresh `m'`. So at `n = 1`, the negative theorem still requires
some `m'` outside `{b₀, b₁}` — which is impossible. Refine the
negative theorem to require `n ≥ 2` (so `|Bitstring n| ≥ 4 > 2`).

### Invariant 1 — Block-sum

**Definition.** For a partition `block : Fin ℓ → Finset (Fin n)`,
`blockSum block b j = (block j).filter (b · = true) |>.card`.

**G-invariance.** A subgroup `G ≤ Equiv.Perm (Fin n)` "preserves
the block partition" iff every `g ∈ G` setwise-permutes each
block: `(block j).image (↑g) = block j`. Under that hypothesis,
`blockSum block` is `G`-invariant.

**Attack.** Apply `invariant_attack` with `f := blockSum block` to
get `∃ A, hasAdvantage` whenever two messages have different
block-sum vectors.

**Defense.** Choosing reps with identical block-sum vectors makes
`IsSeparating (blockSum block) (reps m₀) (reps m₁)` false.

```lean
/-- Per-block bit-count vector. -/
def blockSum {n ℓ : ℕ} (block : Fin ℓ → Finset (Fin n))
    (b : Bitstring n) : Fin ℓ → ℕ :=
  fun j => ((block j).filter (fun i => b i = true)).card

/-- A subgroup `G ≤ S_n` **preserves a block partition** iff every
    `g ∈ G` setwise-permutes each block. -/
def PreservesBlocks {n ℓ : ℕ} (G : Subgroup (Equiv.Perm (Fin n)))
    (block : Fin ℓ → Finset (Fin n)) : Prop :=
  ∀ g : G, ∀ j : Fin ℓ, (block j).image (↑g : Equiv.Perm (Fin n)) = block j

theorem blockSum_invariant_of_preservesBlocks {n ℓ : ℕ}
    (G : Subgroup (Equiv.Perm (Fin n))) (block : Fin ℓ → Finset (Fin n))
    (hPres : PreservesBlocks G block) :
    IsGInvariant (G := ↥G) (blockSum block)

theorem hgoe_blockSum_attack {M : Type*} {n ℓ : ℕ}
    (G : Subgroup (Equiv.Perm (Fin n))) (block : Fin ℓ → Finset (Fin n))
    (hPres : PreservesBlocks G block)
    (scheme : OrbitEncScheme (↥G) (Bitstring n) M) (m₀ m₁ : M)
    (hDiff : blockSum block (scheme.reps m₀) ≠ blockSum block (scheme.reps m₁)) :
    ∃ A : Adversary (Bitstring n) M, hasAdvantage scheme A :=
  invariant_attack scheme (blockSum block) m₀ m₁
    (blockSum_invariant_of_preservesBlocks G block hPres) hDiff

theorem same_blockSum_not_separating {n ℓ : ℕ}
    (block : Fin ℓ → Finset (Fin n))
    (b₀ b₁ : Bitstring n) (hSame : blockSum block b₀ = blockSum block b₁) :
    ¬ IsSeparating (G := Equiv.Perm (Fin n)) (blockSum block) b₀ b₁
```

**G-invariance proof outline.** Show `blockSum block (g • b) j =
blockSum block b j` for every `j`. Unfold:
`((block j).filter ((g • b) ·)).card = ((block j).filter (b (g⁻¹ ·))).card`.
Use `Finset.card_image_of_injective` (with `g⁻¹.toEmbedding`)
combined with `hPres g j : (block j).image g = block j` (so
`(block j).image g⁻¹ = block j` by inversion). The filter on
`(b (g⁻¹ ·))` is the image of the filter on `b` under `g⁻¹`,
hence equicardinal.

### Invariant 2 — Bit parity

**Definition.** `bitParity b := (hammingWeight b) % 2 = 1` (or
equivalently the XOR-fold of all bits). G-invariant under the
**full** `Equiv.Perm (Fin n)` because `hammingWeight` is
S_n-invariant (existing `hammingWeight_invariant`), so `mod 2`
of it is too.

**Attack/defense.** Same `invariant_attack`-instantiation pattern.
Cryptographically: parity is a 1-bit attack channel. A defense
selecting same-parity reps blocks it.

### Invariant 3 — Sorted-bit pattern

**Definition.** The lex-min element of the orbit of `b` under
`Equiv.Perm (Fin n)` (via the existing `bitstringLinearOrder`).

**Decision:** rather than re-prove the structural facts, this

**Proof outline.** `sortedBits_invariant`: by `canon_eq_of_mem_orbit`

**Cryptographic significance.** This is the *strongest* `S_n`-
invariant — it preserves the entire orbit identity. Any attack
mediated by `sortedBits` is exactly the "complete break" via
canonical form. The defense (same-canonical-form reps) is in fact
forbidden by `OrbitEncScheme.reps_distinct`, so this invariant
formalises the **fundamental upper bound** on what `S_n`-invariance
can attack: any pair of distinct messages whose representatives lie
in different orbits is broken by sortedBits. (This connects to the
deterministic OIA vacuity theorem `det_oia_false_of_distinct_reps`
already in `Crypto/OIA.lean`.)

### Edge cases (R-16)

* **`ℓ = 0`** (zero blocks): `blockSum block` is the empty
  function `Fin 0 → ℕ`. Trivially invariant. Defense trivially
  applies (any two messages have "same" block-sum).
* **Single 1-block partition** (`block 0 = Finset.univ`):
  `blockSum block 0 = hammingWeight b`. So R-16's block-sum
  generalises hammingWeight; this is a sanity-check edge case.
* **`n = 0`** (empty bitstrings): `Bitstring 0` is subsingleton,
  all invariants trivially constant.
