## R-05 — Random-oracle / nonce-MAC framework

### Problem

`Orbcrypt/AEAD/CarterWegmanMAC.lean:not_carterWegmanMAC_isQtimeSUFCMASecure`
and the bitstring-polynomial parallel
(`not_bitstringPolynomialMAC_isQtimeSUFCMASecure`) explicitly
construct linear-system attacks that recover the key from any 2
distinct queries — i.e., nonce-free Carter–Wegman is **provably
broken** at Q ≥ 2. The Q-time security gap is acknowledged in
both module docstrings ("nonce-based variant tracked as research
milestone R-05").

The standard cryptographic construction that closes this gap is
Wegman & Carter 1981 §3:
```
tag(k₁, k₂, n, m) := h(k₁, m) + PRF(k₂, n)
```
where `h` is a strongly-universal-2 hash (already proven
ε-SU2 in Orbcrypt) and `PRF` is a pseudo-random function over
nonces. The PRF mask `PRF(k₂, n)` is uniform over `Tag` for each
fresh nonce `n` (under the random-oracle / PRF assumption), so
the tag distribution is uniform conditional on the message and
the joint linear system is degenerate at 1 query per nonce. Hence
**Q-time SUF-CMA reduces to (a) ε-SU2 of `h` plus (b) PRF
indistinguishability of `PRF`**.

This plan formalises the construction inside Lean's posture:
the PRF assumption is a named `Prop` predicate `IsPRF`, the
construction is a structure `NoncedMAC` parametric in
`(h, PRF)`, the Q-time bound is an explicit reduction theorem,
and a non-vacuity witness shows the `Prop` is inhabited at the
ideal random-oracle.

### Mathematical content (verified sound)

**Definition (`IsPRF`).** A function `prf : K → N → Tag` is an
`ε`-PRF iff for every Q-tuple of *distinct* nonces
`(n_1, …, n_Q) : Fin Q ↪ N`, the distinguishing advantage
between the two distributions
* `μ_real := PMF.map (fun k => fun i => prf k (n_seq i))
   (uniformPMF K)` (sample `k` uniformly, evaluate at the Q
   nonces),
* `μ_ideal := uniformPMFTuple Tag Q` (uniform `Tag^Q`),
is at most `ε` for every Boolean distinguisher `D : (Fin Q → Tag)
→ Bool`.

In Lean this is a `Prop`-valued predicate parameterised by `Q : ℕ`
(or quantified over `Q`):

```lean
def IsPRF {K N Tag : Type*} [Fintype K] [Nonempty K]
    [Fintype Tag] [Nonempty Tag] [DecidableEq N]
    (prf : K → N → Tag) (Q : ℕ) (ε : ℝ≥0∞) : Prop :=
  ∀ (n_seq : Fin Q ↪ N) (D : (Fin Q → Tag) → Bool),
    advantage D
      (PMF.map (fun k => fun i => prf k (n_seq i)) (uniformPMF K))
      (uniformPMFTuple Tag Q)
      ≤ ε.toReal
```

**Equivalent reformulation (used by the reduction).** For Q
distinct nonces, the joint distribution `(prf k n_1, …, prf k n_Q)`
under uniform `k` is at advantage ≤ ε from the uniform `Tag^Q`
distribution; equivalently, for every `(t_1, …, t_Q) ∈ Tag^Q`,
`|μ_real(t_1, …, t_Q) − 1/|Tag|^Q| ≤ ε / |Tag|^Q` (at the cost
of a `|Tag|^Q` multiplier; we'll use the `advantage`-form to
avoid that).

**Theorem 1 (Q-time SUF-CMA reduction; nonce-form Wegman–Carter).**
Let `h : K_h → Msg → Tag` be ε_h-SU2 (in the sense of
`IsEpsilonSU2` from `Probability/UniversalHash.lean`), and let
`prf : K_p → N → Tag` be ε_p-PRF (in the sense above) at
Q-tuples. Define
```
noncedMAC.tag (k₁, k₂) (n, m) := h k₁ m + prf k₂ n
noncedMAC.verify (k₁, k₂) (n, m) t := decide (t = h k₁ m + prf k₂ n)
```
Then for every Q-time non-adaptive nonce-respecting adversary
`A` (each query has a fresh nonce; the forge's nonce may or may
not be fresh), `forgeryAdvantage_Qtime noncedMAC A ≤ ε_h + ε_p`.

**Proof sketch (full discharge below in Phase 3).**

1. **Hybrid 1: ideal PRF.** Replace `prf k₂ n_i` with truly
   uniform `s_i ∈ Tag`. By `IsPRF`, the adversary's view changes
   by at most `ε_p` (this is the standard PRF→RO step).

2. **Hybrid 2: 1-time SU2 reduction.** In Hybrid 1, for each
   query `(n_i, m_i)`, the tag is `t_i := h k₁ m_i + s_i` with
   `s_i ~ Uniform Tag`. The marginal of `t_i` is uniform
   over `Tag` (since `s_i` is uniform and additive masking by
   uniform is uniform). The conditional distribution of `t_i`
   given `(t_1, …, t_{i-1})` is also uniform (the `s_j` are
   independent across distinct nonces).

3. **Forge analysis.** The forge `(n_forge, m_forge, t_forge)`
   is required to satisfy `n_forge ∉ {n_1, …, n_Q}` ∨
   `m_forge ∉ {m_1, …, m_Q}` (SUF-CMA freshness). Two cases:

   * **Fresh nonce.** `s_forge ~ Uniform Tag` is independent of
     the adversary's view. The win event `t_forge = h k₁ m_forge
     + s_forge` is satisfied with probability exactly `1/|Tag|`
     (over the uniform `s_forge`).
   * **Reused nonce, fresh message.** WLOG `n_forge = n_i` for
     some `i ≤ Q`. The mask `s_i` is *known* (the adversary saw
     `t_i = h k₁ m_i + s_i`). The forge equation becomes
     `t_forge = h k₁ m_forge + (t_i − h k₁ m_i)`, equivalently
     `h k₁ m_forge − h k₁ m_i = t_forge − t_i`. By ε_h-SU2 (the
     joint-distribution bound at messages `(m_i, m_forge)` and
     tag pair `(t_i, t_forge − (t_forge − t_i))` … see the
     careful derivation in Phase 3), the probability is ≤ ε_h.

4. **Sum.** Total advantage in Hybrid 2 is ≤ `max(1/|Tag|, ε_h)`
   per forge attempt. Combined with the Hybrid 1 substitution
   cost ε_p, the total bound is `ε_h + ε_p`. (When ε_h ≥ 1/|Tag|,
   the `max` is ε_h; in the ε_h < 1/|Tag| regime — possible for
   bitstring-polynomial at large `p` — we use the looser
   `1/|Tag| ≤ ε_h + 1/|Tag|` bound, sometimes written as
   `ε_h + ε_p + 1/|Tag|`. The plan's headline bound is the
   simpler `ε_h + ε_p` form, with the `1/|Tag|` term absorbed
   into ε_h via a `max` step when relevant.)

**Concrete ε for `nonceCarterWegmanMAC`.** With `h =
carterWegmanHash p` (ε_h = 1/p) and any ε_p-PRF, the
Q-time SUF-CMA advantage is ≤ `1/p + ε_p`. At the ideal
PRF (`ε_p = 0`), this is `1/p` independently of Q —
quantitatively *better* than the Q-time NEGATIVE result of
the nonce-free MAC (which has advantage 1).

**Concrete ε for `nonceBitstringPolynomialMAC`.** With `h =
bitstringPolynomialHash p n` (ε_h = n/p) and any ε_p-PRF, the
Q-time SUF-CMA advantage is ≤ `n/p + ε_p`. At the ideal PRF,
this is `n/p` independently of Q.

**Non-vacuity at the ideal random-oracle.** The truly-random
oracle `prf := fun (k : (N → Tag)) (n : N) => k n` (where the
"key" is itself a function, sampled uniformly from `N → Tag`)
trivially satisfies `IsPRF prf Q 0` for every `Q` because the
two distributions `μ_real` and `μ_ideal` *coincide* — sampling
`k : N → Tag` uniformly and evaluating at Q distinct nonces
produces a uniform `Tag^Q` sample. The non-vacuity witness for
`IsPRF` therefore lands at ε = 0 with `K = (N → Tag)`,
demonstrating the predicate is inhabited and the reduction
theorem is non-vacuously applicable.

### Design decisions

* **PRF abstraction over `ε`.** `IsPRF prf Q ε` carries an
  *explicit* ε bound rather than collapsing to "ε = 0 ideal" or
  "negligible asymptotic". This matches Orbcrypt's existing
  `IsEpsilonUniversal` / `IsEpsilonSU2` / `ConcreteOIA` style:
  ε-explicit predicates compose cleanly with the existing
  `concrete_*` chain, and downstream consumers can plug in any
  concrete bound (e.g., `ε = 2^{-128}` for AES-CTR-PRF used as a
  PRF, modulo the AES PRF assumption).
* **Q parameter ON the predicate, not universally quantified.**
  `IsPRF prf Q ε` is `Q`-specific so the reduction theorem can
  consume the *specific Q* the adversary makes queries at, rather
  than a universal-Q bound. This matches the standard
  cryptographic literature (Q-tuple PRF security; cf.
  Bellare-Rogaway "The Security of Triple Encryption" 2006 §3).
  Universal-Q PRF (uniform across all Q) is recoverable as
  `∀ Q, IsPRF prf Q ε` and is provided as a corollary
  `IsPRF.uniformQ` for callers that want it.
* **No `noncomputable` on the structure.** `NoncedMAC` is a plain
  structure; the `tag` and `verify` fields are `def`s, not
  `noncomputable`. (The PMF-monad reasoning happens inside
  `IsPRF` and the reduction proof, not in the construction
  itself.) This keeps `nonceCarterWegmanMAC`, etc., usable in
  `decide`-backed audit-script `example` bindings (parallel to
  the existing `carterWegmanMAC`).
* **`NoncedMAC` is *not* a `MAC` instance.** A `MAC` (per
  `AEAD/MAC.lean`) takes `(K, Msg, Tag)` and tags messages
  directly. A `NoncedMAC` takes `(K_h × K_p, N × Msg, Tag)` and
  tags `(nonce, message)` pairs. They are structurally distinct
  abstractions because the SUF-CMA game shape differs (nonce
  freshness vs. message freshness). The plan keeps them
  separate: `NoncedMAC` has its own structure + own SUF-CMA
  game `IsQtimeSUFCMASecureNonced` parametric in nonce-freshness
  semantics. A bridging `NoncedMAC.toMAC` adapter (treating the
  nonce as part of the message) is intentionally NOT introduced
  because it would silently break the freshness game shape.
* **Adaptive vs. non-adaptive Q-time queries.** The headline
  reduction `noncedMAC_isQtimeSUFCMASecure_of_isPRF` is stated
  for **non-adaptive** Q-time adversaries (the `MultiQueryMACAdversary`
  shape from `AEAD/MACSecurity.lean`). Adaptive Q-time adversaries
  would require a oracle-style game; the standard hybrid argument
  upgrades non-adaptive to adaptive at no advantage cost when the
  PRF is adaptive-secure, but Lean-level oracle abstractions are
  out of scope for this PR. Documented as research-scope follow-up
  R-05⁺ (adaptive Q-time = full SUF-CMA-2).
* **Nonce-freshness convention.** The Q-time game requires
  *distinct* nonces across the Q queries (the standard
  "nonce-respecting" convention from Rogaway). The adversary may
  reuse the forge's nonce (i.e., `n_forge` may equal one of
  `n_1, …, n_Q`) but must produce a `(n_forge, m_forge, t_forge)`
  with either `n_forge` fresh OR `m_forge` fresh among the queried
  `(n_i, m_i)` pairs. This matches Rogaway's "deterministic
  authenticated encryption" SUF-CMA-2 game (Rogaway 2004, Eurocrypt).

### New declarations

```lean
/-- A **nonced MAC** carries a hash family `hash : K_h → Msg → Tag`
    and a PRF `prf : K_p → Nonce → Tag`. The tag of a `(nonce,
    message)` pair is `hash k_h m + prf k_p n` where `+` is the
    additive structure on `Tag`. -/
structure NoncedMAC (K_h K_p Nonce Msg Tag : Type*) where
  hash : K_h → Msg → Tag
  prf  : K_p → Nonce → Tag

/-- The tag function. Combines the hash of the message with the
    PRF mask of the nonce by additive masking. -/
def NoncedMAC.tag {K_h K_p Nonce Msg Tag : Type*} [Add Tag]
    (mac : NoncedMAC K_h K_p Nonce Msg Tag)
    (k_h : K_h) (k_p : K_p) (n : Nonce) (m : Msg) : Tag :=
  mac.hash k_h m + mac.prf k_p n

/-- Verification: equality check against the regenerated tag. -/
def NoncedMAC.verify {K_h K_p Nonce Msg Tag : Type*}
    [Add Tag] [DecidableEq Tag]
    (mac : NoncedMAC K_h K_p Nonce Msg Tag)
    (k_h : K_h) (k_p : K_p) (n : Nonce) (m : Msg) (t : Tag) : Bool :=
  decide (t = mac.tag k_h k_p n m)

/-- A non-adaptive Q-time nonced-MAC adversary: commits to Q
    `(nonce, message)` queries, then forges given the Q tags. The
    nonce-respecting convention is enforced inside `forges`: the
    `queries`'s nonces must be pairwise distinct. -/
structure NoncedMultiQueryMACAdversary
    (K_h K_p Nonce Msg Tag : Type*) (Q : ℕ) where
  queries : Fin Q → Nonce × Msg
  forge   : (Fin Q → Tag) → Nonce × Msg × Tag

/-- The adversary forges iff the forge verifies under the keys AND
    is fresh (the `(n_forge, m_forge)` pair is not among the
    queried pairs). -/
def NoncedMultiQueryMACAdversary.forges
    {K_h K_p Nonce Msg Tag : Type*} {Q : ℕ}
    [Add Tag] [DecidableEq Tag] [DecidableEq Nonce] [DecidableEq Msg]
    (mac : NoncedMAC K_h K_p Nonce Msg Tag)
    (A : NoncedMultiQueryMACAdversary K_h K_p Nonce Msg Tag Q)
    (k_h : K_h) (k_p : K_p) : Bool :=
  let tags : Fin Q → Tag :=
    fun i => mac.tag k_h k_p (A.queries i).1 (A.queries i).2
  let (n_forge, m_forge, t_forge) := A.forge tags
  -- nonce-respecting: queried nonces distinct
  let queries_distinct : Bool :=
    decide (Function.Injective (fun i => (A.queries i).1))
  -- fresh-(nonce, message): the forge's pair is not among the queries
  let fresh : Bool :=
    decide (∀ i : Fin Q, (n_forge, m_forge) ≠ A.queries i)
  queries_distinct && fresh && mac.verify k_h k_p n_forge m_forge t_forge

/-- Q-time SUF-CMA forgery advantage. -/
noncomputable def noncedForgeryAdvantage_Qtime
    {K_h K_p Nonce Msg Tag : Type*} {Q : ℕ}
    [Fintype K_h] [Fintype K_p] [Nonempty K_h] [Nonempty K_p]
    [Add Tag] [DecidableEq Tag] [DecidableEq Nonce] [DecidableEq Msg]
    (mac : NoncedMAC K_h K_p Nonce Msg Tag)
    (A : NoncedMultiQueryMACAdversary K_h K_p Nonce Msg Tag Q) : ℝ :=
  (probTrue ((uniformPMF K_h).prod (uniformPMF K_p))
    (fun (k : K_h × K_p) => A.forges mac k.1 k.2)).toReal

/-- A nonced MAC is `ε`-Q-time-SUF-CMA-secure iff the forgery
    advantage of every Q-time non-adaptive adversary is ≤ ε. -/
def IsNoncedQtimeSUFCMASecure
    {K_h K_p Nonce Msg Tag : Type*} {Q : ℕ}
    [Fintype K_h] [Fintype K_p] [Nonempty K_h] [Nonempty K_p]
    [Add Tag] [DecidableEq Tag] [DecidableEq Nonce] [DecidableEq Msg]
    (mac : NoncedMAC K_h K_p Nonce Msg Tag) (ε : ℝ) : Prop :=
  ∀ (A : NoncedMultiQueryMACAdversary K_h K_p Nonce Msg Tag Q),
    noncedForgeryAdvantage_Qtime mac A ≤ ε

/-- A function `prf : K_p → Nonce → Tag` is an `ε`-PRF on
    `Q`-tuples of distinct nonces iff for every injective Q-tuple
    `nonces : Fin Q ↪ Nonce` and every Boolean distinguisher
    `D : (Fin Q → Tag) → Bool`, the advantage between the joint
    real-PRF distribution and the uniform `Tag^Q` distribution is
    at most ε. -/
def IsPRF {K_p Nonce Tag : Type*}
    [Fintype K_p] [Nonempty K_p]
    [Fintype Tag] [Nonempty Tag] [DecidableEq Nonce]
    (prf : K_p → Nonce → Tag) (Q : ℕ) (ε : ℝ≥0∞) : Prop :=
  ∀ (nonces : Fin Q → Nonce) (D : (Fin Q → Tag) → Bool),
    Function.Injective nonces →
    advantage D
      (PMF.map (fun k_p : K_p => fun i => prf k_p (nonces i))
        (uniformPMF K_p))
      (uniformPMFTuple Tag Q) ≤ ε.toReal

/-- Monotonicity in ε. -/
theorem IsPRF.mono {prf : K_p → Nonce → Tag} {Q : ℕ} {ε ε' : ℝ≥0∞}
    (h : IsPRF prf Q ε) (hε : ε ≤ ε') : IsPRF prf Q ε'

/-- Every prf is trivially `1`-PRF (advantage ≤ 1 always). -/
theorem IsPRF.le_one (prf : K_p → Nonce → Tag) (Q : ℕ) :
    IsPRF prf Q 1

/-- **Truly-random-oracle PRF.** The "key" is the entire random
    function `Nonce → Tag`; the PRF is just `fun k n => k n`.
    Sampling `k` uniformly from `Nonce → Tag` and projecting at Q
    distinct nonces gives a uniform `Tag^Q` sample, so the
    advantage between real and ideal PMF is exactly 0. -/
def idealRandomOraclePRF (Nonce Tag : Type*) :
    (Nonce → Tag) → Nonce → Tag :=
  fun k n => k n

/-- The truly-random oracle is a `0`-PRF for every Q. The
    pushforward `PMF.map (fun k => fun i => k (nonces i))
    (uniformPMF (Nonce → Tag))` equals `uniformPMFTuple Tag Q`
    *exactly* — not approximately — when `nonces` is injective. -/
theorem idealRandomOraclePRF_isPRF
    [Fintype Nonce] [Fintype Tag] [Nonempty Tag]
    [DecidableEq Nonce] [DecidableEq Tag] (Q : ℕ) :
    IsPRF (idealRandomOraclePRF Nonce Tag) Q 0

/-- **Headline R-05 reduction.** A nonced MAC is Q-time SUF-CMA-
    secure under (a) ε_h-AXU on the hash and (b) ε_p-PRF security
    on the PRF for `(Q + 1)`-tuples (the Q queries plus the
    forgery nonce). The advantage bound is
    `Q · ε_h + ε_p + 1/|Tag|`.

    The bound shape comes from a 2-step hybrid:
    * Hybrid 1 substitutes the PRF with a truly-random function;
      cost ε_p (PRF assumption).
    * Hybrid 2 analyses the random-mask game: re-used-nonce
      forgeries cost ≤ ε_h each (Q nonces ⇒ Q · ε_h); fresh-nonce
      forgeries cost ≤ 1/|Tag| (uniform mask).
-/
theorem noncedMAC_isQtimeSUFCMASecure_of_isAXU_and_isPRF
    {K_h K_p Nonce Msg Tag : Type*} {Q : ℕ}
    [Fintype K_h] [Fintype K_p] [Nonempty K_h] [Nonempty K_p]
    [Fintype Tag] [Nonempty Tag]
    [DecidableEq K_h] [DecidableEq K_p]
    [DecidableEq Msg] [DecidableEq Tag] [DecidableEq Nonce]
    [AddCommGroup Tag]
    (mac : NoncedMAC K_h K_p Nonce Msg Tag)
    (ε_h ε_p : ℝ≥0∞) (h_h_finite : ε_h ≠ ⊤) (h_p_finite : ε_p ≠ ⊤)
    (hAXU : IsEpsilonAXU mac.hash ε_h)
    (hPRF : IsPRF mac.prf (Q + 1) ε_p) :
    IsNoncedQtimeSUFCMASecure (Q := Q) mac
      ((Q : ℝ≥0∞) * ε_h + ε_p +
        ((Fintype.card Tag : ℝ≥0∞)⁻¹)).toReal

/-- Specialisation: `nonceCarterWegmanMAC p` composes
    `carterWegmanHash p` with the truly-random oracle on
    `ZMod p × ℕ`-keyed PRFs. -/
def nonceCarterWegmanMAC (p : ℕ) [Fact (Nat.Prime p)] :
    NoncedMAC (ZMod p × ZMod p) (ℕ → ZMod p) ℕ (ZMod p) (ZMod p) where
  hash := carterWegmanHash p
  prf  := idealRandomOraclePRF ℕ (ZMod p)

/-- Specialisation: `nonceBitstringPolynomialMAC p n` composes
    `bitstringPolynomialHash p n` with the truly-random oracle. -/
def nonceBitstringPolynomialMAC (p n : ℕ) [Fact (Nat.Prime p)] :
    NoncedMAC (ZMod p × ZMod p) (ℕ → ZMod p) ℕ (Bitstring n) (ZMod p) where
  hash := bitstringPolynomialHash p n
  prf  := idealRandomOraclePRF ℕ (ZMod p)

/-- **Ideal-PRF + Carter-Wegman SUF-CMA bound.** At the truly-
    random oracle (ε_p = 0) and Carter-Wegman ε_h = 1/p, the
    Q-time SUF-CMA advantage is `Q/p + 1/p` — independent of Q
    only up to the linear `Q` factor. -/
theorem ideal_nonceCarterWegmanMAC_isQtimeSUFCMASecure
    (p : ℕ) [Fact (Nat.Prime p)] (Q : ℕ) :
    IsNoncedQtimeSUFCMASecure (Q := Q) (nonceCarterWegmanMAC p)
      (((Q + 1 : ℝ≥0∞) / p)).toReal

/-- **Status disclosure for R-05.**
    Unconditional (machine-checked):
    * `idealRandomOraclePRF` is `0`-PRF for every Q.
    * `nonceCarterWegmanMAC` and `nonceBitstringPolynomialMAC`
      achieve `(Q+1)/p` and `(Q·n + 1)/p` Q-time SUF-CMA
      respectively, AT the truly-random oracle.

    Research-scope (R-05⁺):
    * Concrete instantiations with non-ideal PRFs (HMAC, AES-CTR
      as PRF). Discharging `IsPRF` for these requires the
      corresponding cryptographic assumption (HMAC-PRF, AES-PRF)
      which is not provable inside Lean.
    * Adaptive Q-time queries (full SUF-CMA-2 / oracle access).
      Requires Lean-level oracle-game abstractions.
-/
theorem r05_research_scope_disclosure :
    IsPRF (idealRandomOraclePRF ℕ (ZMod 2)) 1 0 :=
  idealRandomOraclePRF_isPRF 1
```

**Phase 3 — Q-time SUF-CMA reduction** (~280 LOC, 4.5 days,
`Orbcrypt/AEAD/NoncedMACSecurity.lean`).
This is the most complex phase; broken into 9 sub-units.

**Phase 3 proof outline (verified-sound).** The reduction is a
two-step game-hopping argument:

```
Game G0 (real game)              cost   advantage
  k_h ← K_h, k_p ← K_p             —     real-PRF view
  (queries, forge_fn) ← A
  tags := tag(k_h, k_p, q_i)
  (n*, m*, t*) := forge_fn(tags)
  Win = fresh ∧ verify(k_h, k_p, n*, m*, t*)

Game G1 (ideal-PRF substitution)  cost   ≤ ε_p
  k_h ← K_h, s : Fin (Q+1) → Tag (uniform)
  tags_i := hash(k_h, q_i.2) + s(query_idx_i)
  (n*, m*, t*) := forge_fn(tags)
  Win = fresh ∧ (t* = hash(k_h, m*) + s(forge_idx))
  -- where query_idx_i = i (the i-th query) and
  -- forge_idx = either i (re-use n*=q_i.1) or Q (fresh).
```

The `IsPRF` hypothesis at `Q+1` queries says the joint PMF of
(s_1, …, s_{Q+1}) is at advantage ≤ ε_p from uniform. The
distinguisher we wrap is "given (s_1, …, s_{Q+1}), simulate G1
and return the win bit". Game G1's win event probability is
within ε_p of Game G0's.

In Game G1, conditional on the query indices and the
forge_idx, the win probability decomposes:

* **Fresh-nonce branch (forge_idx = Q):** `s(Q)` is uniform and
  *independent* of the adversary's view (the view depends on
  `s(0), …, s(Q-1)` only). Hence
  `Pr[t* = hash(k_h, m*) + s(Q)] = 1/|Tag|`.
* **Re-used-nonce branch (forge_idx = i for some i < Q):** the
  adversary knows `tags_i = hash(k_h, q_i.2) + s(i)`, hence
  `s(i) = tags_i - hash(k_h, q_i.2)`. The forge's verification
  equation becomes:
  ```
  t* = hash(k_h, m*) + s(i)
     = hash(k_h, m*) + tags_i - hash(k_h, q_i.2)
  ```
  Equivalently, `hash(k_h, m*) - hash(k_h, q_i.2) = t* - tags_i`.
  By `IsEpsilonAXU` of `hash` at distinct messages
  `(m*, q_i.2)` (distinct because of the freshness hypothesis
  `(n*, m*) ≠ (q_i.1, q_i.2)`; if `n* = q_i.1` then `m* ≠
  q_i.2`), this equality holds with probability ≤ ε_h over
  uniform `k_h`.

The forge_idx is determined by the adversary's choice of `n*`
(it's one of Q+1 possible values: `{q_0.1, …, q_{Q-1}.1, n*}`
when `n*` is fresh; or one of `q_i.1` when `n* = q_i.1`).
Summing over the Q possible re-use indices: `Q · ε_h`. Plus
the fresh-nonce contribution `1/|Tag|` (when forge_idx = Q).

**Total bound: G0 advantage ≤ G1 advantage + ε_p ≤ Q · ε_h +
1/|Tag| + ε_p**.

**Phase 3 work units.**

* **WU-3.1 (~30 LOC) — Win event decomposition.** Define
  `winAtNonceIdx : (Fin Q → Tag) → Fin (Q + 1) → Bool` that
  classifies the forge's nonce by `Q+1` cases (Q reuse-indices
  + 1 fresh). The total win event is the disjoint union over
  the `Q+1` cases. Helper: `forgeNonceIdx` extracting the case
  from the forge's nonce vs the queries.
* **WU-3.2 (~30 LOC) — Probability factorisation.** Show
  `Pr[Win] = Σ_{i : Fin (Q+1)} Pr[Win ∧ forgeNonceIdx = i]`.
  Uses the fact that `winAtNonceIdx` partitions the forge's
  nonce.
* **WU-3.3 (~25 LOC) — Hybrid setup.** Define `gameG1` PMF
  that samples `s : Fin (Q + 1) → Tag` uniformly instead of via
  `prf`. Define the simulating distinguisher `D_PRF : (Fin (Q +
  1) → Tag) → Bool` that runs A's forge under the s-sampled
  tags and returns the win bit.
* **WU-3.4 (~50 LOC) — Hybrid step (real → ideal).** Apply
  `IsPRF prf (Q + 1) ε_p` to `D_PRF` to get
  `|Pr_G0[Win] - Pr_G1[Win]| ≤ ε_p`. The PRF query nonces are
  the queried `q_i.1` for `i < Q` plus the forge's `n*` (which
  may or may not be in the query set). When `n*` is reused,
  the (Q+1)-th nonce equals one of the first Q; the
  injectivity hypothesis on `nonces` is satisfied by the
  augmented nonces `q_0.1, q_1.1, …, q_{Q-1}.1, n*` ONLY when
  `n*` is fresh. For the re-use case, IsPRF is applied at Q
  nonces (not Q+1) — handled by case-splitting on freshness
  before applying IsPRF.
* **WU-3.5 (~30 LOC) — Fresh-nonce branch in G1.** Show
  `Pr_G1[Win ∧ forge_idx = Q] ≤ 1/|Tag|`. Argument: in G1,
  the (Q+1)-th mask `s(Q)` is uniformly distributed
  *independently* of `(s(0), …, s(Q-1), k_h)` (which the
  adversary's view depends on). The win equation `t* =
  hash(k_h, m*) + s(Q)` uniquely determines a single value
  of `s(Q)` for each `(k_h, m*, t*)` triple; uniform mass at
  that value is `1/|Tag|`.
* **WU-3.6 (~50 LOC) — Re-used-nonce branch in G1.** Show
  `∀ i : Fin Q, Pr_G1[Win ∧ forge_idx = i] ≤ ε_h / Q` (bound
  per re-use index; sum is `Q · (ε_h / Q) = ε_h`). Wait — the
  per-index bound is actually `ε_h` (each is bounded by ε_h
  independently); summing gives `Q · ε_h`. Argument: at
  forge_idx = i, the win equation reduces to
  `hash(k_h, m*) - hash(k_h, q_i.2) = t* - tags_i`. The
  randomness is over `k_h` (uniform) and `s(j)` for `j ≠ i`
  (which don't affect the equation). The conditional win
  probability over uniform `k_h` is bounded by ε_h via
  IsEpsilonAXU at messages `(m*, q_i.2)` (distinct by
  freshness). Sum over Q values of i: `Q · ε_h`.
* **WU-3.7 (~30 LOC) — Sum across cases.** Combine WU-3.5 +
  WU-3.6: `Pr_G1[Win] ≤ Q · ε_h + 1/|Tag|`. With WU-3.4:
  `Pr_G0[Win] ≤ Pr_G1[Win] + ε_p ≤ Q · ε_h + 1/|Tag| + ε_p`.
* **WU-3.8 (~25 LOC) — `.toReal` and ENNReal arithmetic
  finalisation.** Convert the ENNReal bound to ℝ via
  `ENNReal.toReal_add` and the finiteness hypotheses
  `h_h_finite`, `h_p_finite`. Output the final
  `IsNoncedQtimeSUFCMASecure` shape.
* **WU-3.9 (~10 LOC) — `_one_one` regression sentinel.** At
  `ε_h = 1` and `ε_p = 1`, the bound is `Q + 1 + 1/|Tag| ≤
  Q + 2` (trivially satisfiable; covered by `_le_one`
  helpers).

### Common pitfalls (R-05)

* **`uniformPMFTuple` vs `(uniformPMF α)^Q`.** The two are
  equal but Mathlib's signature uses `Fin Q → α` not
  `(α)^Q`. Always thread through `uniformPMFTuple`; do not
  attempt manual product PMF construction.
* **`PMF.bind_apply` vs `PMF.map_apply`.** `bind` is
  sequential composition with PMF-`g`; `map` is pushforward
  through a function. The IsPRF predicate uses `map`; the
  reduction proof uses `bind` for the game's randomness
  threading.
* **`Function.Injective` on `Fin Q → Nonce`.** When `Q > |Nonce|`,
  the predicate is unsatisfiable (pigeonhole). The IsPRF
  hypothesis only quantifies over injective `Fin Q → Nonce`,
  so vacuously satisfied at `Q > |Nonce|`. Document this in the
  IsPRF docstring.
* **`AddCommGroup Tag` vs `Add Tag`.** The `tag` and `verify`
  functions need only `[Add Tag]`. The reduction proof needs
  `[AddCommGroup Tag]` for the inverse-mask manipulation
  (`s(i) = tags_i - hash(k_h, q_i.2)`). Thread the stronger
  typeclass at the theorem level only.
* **`s_finite` typeclass on `Fintype Tag`.** The `Fintype Tag`
  typeclass is required for `uniformPMF Tag` to make sense.
  But the IsPRF predicate uses `Tag := ZMod p` for concrete
  witnesses, where `[Fintype (ZMod p)]` is automatic.
* **The "fresh-nonce" branch's independence argument.** The
  adversary's view in G1 depends on `s(0), …, s(Q-1)`; the
  fresh-nonce mask `s(Q)` is independent. This independence is
  syntactic (the `s(Q)` projection commutes with the
  view-extraction function); discharged by
  `Function.const_apply` and `PMF.map_const`.

### Edge cases (R-05)

* **Q = 0.** The Q-time game with no queries is vacuously satisfied
  (the adversary commits to no queries, can only forge a fresh
  `(n_forge, m_forge)`). The reduction theorem reduces to the
  fresh-nonce branch, giving `0 · ε_h + ε_p + 1/|Tag| = ε_p +
  1/|Tag|`. At the ideal oracle (ε_p = 0), this is `1/|Tag|`.
  Documented in the headline docstring.
* **|Tag| = 1.** The `1/|Tag|` term collapses to `1`, so the bound
  is `≥ 1` and the theorem is trivially satisfied. Documented in
  the headline docstring as a degenerate-case note.
* **Empty `Nonce`.** `[Nonempty Nonce]` is required indirectly via
  `Function.Injective nonces : Fin Q → Nonce` for Q ≥ 1; at Q = 0
  the predicate quantifies vacuously and no Nonce is needed.
* **Adaptive vs non-adaptive.** The plan delivers non-adaptive
  Q-time only. Adaptive Q-time (oracle access) requires a different
  game shape and is research-scope R-05⁺.
* **Reused nonces (game-shape violation).** The adversary's
  `queries` may have repeated nonces; the `forges` function rejects
  this via the `queries_distinct` check. This matches the
  "nonce-respecting adversary" convention from Rogaway 2004.

## R-11 — Concrete non-trivial `CommGroupAction` instance

### Problem

`Orbcrypt/PublicKey/CommutativeAction.lean:81-83` defines the
`CommGroupAction` typeclass for commutative group actions
underlying CSIDH-style key exchange. The only currently-registered
instance is `selfAction` (line ~280): a `CommGroup G` acting on
itself by left multiplication — which is commutative but is
**broken in polynomial time** by the discrete-log-in-the-group
algorithm whenever `G` is cyclic of large prime order. The module
docstring (lines 30-46) acknowledges this gap explicitly:
"concrete commutative actions with the required hardness
properties are the scarce resource".

The full CSIDH primitive (ideal class group action on
supersingular elliptic curves) is multi-year research-scope work
because it requires class-field-theory + isogeny formalisation in
Mathlib (none of which is currently available). **However, a
substantial cryptographic-content-bearing intermediate step is
available**: the *multiplicative-group action* of `(ZMod p)ˣ` on
`ZMod p`. This action is:

* **Genuinely non-trivial** — orbits are proper subsets of the
  ambient space (`{0}` is fixed; `(ZMod p)ˣ` is the unique non-zero
  orbit, of cardinality p − 1).
* **Commutative** — `(ZMod p)ˣ` is abelian.
* **Backed by a named hardness assumption** — the discrete
  logarithm problem in `(ZMod p)ˣ`, AKA the "computational
  Diffie–Hellman" assumption. Decisional DH (DDH) is the standard
  IND-CPA hypothesis for this group.

R-11 lands the `CommGroupAction` typeclass instance for
`multGroupCommAction p`, an `IsCommActionDDHHard` Prop
predicate capturing the DDH assumption, and an IND-CPA-style
reduction theorem for `CommOrbitPKE` conditional on the Prop.
The DDH Prop itself is not proved (it is a standard
cryptographic assumption); concrete witnesses inhabit the Prop
trivially at ε = 1 (`advantage_le_one`) and via the toy
`(ZMod 7)ˣ`-instance for non-vacuity testing.

### Mathematical content (verified sound)

**Multiplicative-group action.** For prime `p`, the units
`(ZMod p)ˣ` form a cyclic group of order `p − 1`. The action
`u • x := (u : ZMod p) * x` on the full residue ring `ZMod p`
satisfies:
* **Identity:** `(1 : (ZMod p)ˣ) • x = (1 : ZMod p) * x = x`.
* **Compatibility:** `(u * v) • x = (u v : ZMod p) * x =
  (u : ZMod p) * ((v : ZMod p) * x) = u • (v • x)`.
* **Commutativity:** `u • (v • x) = u v x = v u x = v • (u • x)`
  by commutativity of multiplication in `ZMod p`.

The orbits are:
* `{0}` (the zero residue is fixed by every unit).
* `(ZMod p) \ {0}` (a single orbit covering all non-zero
  residues; transitive because `(ZMod p)ˣ` acts simply
  transitively on the units when `p` is prime).

**`CommOrbitPKE` instantiation.** Take `basePoint := 1 : ZMod p`
(any non-zero residue works; we pick `1` for canonical-ness).
Secret key `secretKey ∈ (ZMod p)ˣ`. Public key `publicKey =
(secretKey : ZMod p) * 1 = (secretKey : ZMod p)`. The
`pk_valid` field requires `publicKey = secretKey • basePoint`,
which is `rfl` after `mul_one`.

**Encryption.** `encrypt (pk, x₀) m` samples `r ←$ (ZMod p)ˣ`,
returns `(r • basePoint, r • publicKey)` — the standard ElGamal
shape adapted to the action layer. The first component is the
ephemeral half-key Alice transmits; the second is the encrypted
shared secret.

**Decryption.** `decrypt (sk, c) := sk • c.1`. Since `sk` and
`r` commute, `sk • c.1 = sk • (r • basePoint) = r • (sk •
basePoint) = r • publicKey = c.2`, so the decryption recovers
the shared secret.

**`IsCommActionDDHHard` predicate.** No PPT adversary
distinguishes `(basePoint, sk • basePoint, r • (sk • basePoint))`
from `(basePoint, sk • basePoint, t • basePoint)` for uniform
`r, sk, t`, with advantage > ε. Captured as a `Prop` predicate
parameterised by ε:

```lean
def IsCommActionDDHHard
    [Group G] [CommGroupAction G X] [Fintype G] [Nonempty G]
    [DecidableEq X]
    (bp : X) (ε : ℝ) : Prop :=
  ∀ (D : X → X → X → Bool),
    advantage (fun (t : G × G × G) => D bp (t.1 • bp) (t.2.2 • bp))
      (PMF.map (fun (sk r : G) => (sk, r, sk * r))
         ((uniformPMF G).prod (uniformPMF G)))
      (PMF.map (fun (sk r t : G) => (sk, r, t))
         ((uniformPMF G).prod ((uniformPMF G).prod (uniformPMF G))))
      ≤ ε
```

(Concrete shape will be refined to fit the existing
`advantage` signature in `Probability/Advantage.lean` —
two-argument PMF distinguishing — by encoding the triple into
`X × X × X` directly.)

**IND-CPA reduction.** Standard ElGamal/textbook DH-PKE
argument: any IND-CPA distinguisher gives a DDH distinguisher
by simulating the IND-CPA game. The reduction is parametric
in ε: an IND-CPA advantage of γ produces a DDH-distinguishing
advantage ≥ γ, contradicting `IsCommActionDDHHard ε` whenever
γ > ε. Equivalently, IND-CPA advantage ≤ ε under
`IsCommActionDDHHard ε`.

### Design decisions

* **Two-module split: hardness Prop vs concrete instance.**
  The hardness Prop module (`CSIDHHardness.lean`) is parametric
  over any `CommGroupAction`; the concrete instance module
  (`MultGroupAction.lean`) plugs in `(ZMod p)ˣ`. This lets future
  research-scope work (full CSIDH on supersingular curves) drop
  in a parallel `SupersingularIsogenyAction.lean` module without
  re-deriving the IND-CPA reduction.
* **`(ZMod p)ˣ` over `ZMod p` (not `selfAction`).** The
  `selfAction` (a `CommGroup G` acting on itself) is already
  registered; R-11's content is the *non-trivial* action where
  the acting group is a strict subset of the carrier. The
  multiplicative-group action is the canonical Mathlib-friendly
  example: `(ZMod p)ˣ ↪ ZMod p` is a Mathlib coercion
  (`Units.coe_one`, `Units.coe_mul`).
* **Fixed-point handling.** The orbit of `0 : ZMod p` is `{0}`
  (singleton, trivially distinguishable). The orbit of any
  non-zero residue is `(ZMod p) \ {0}` (the units). The plan
  picks `basePoint := 1` and the IND-CPA reduction is stated
  for non-fixed-point basepoints; a separate
  `multGroupAction_orbit_of_one` lemma characterises the orbit
  exactly.
* **Adversary advantage formalism.** R-11 reuses the `advantage`
  definition from `Probability/Advantage.lean`. The DDH game's
  triple structure `(bp, sk•bp, c)` is encoded into a single
  PMF over `X × X × X`; the distinguisher is a Boolean function
  on triples. This matches the existing `advantage` two-PMF
  shape after currying.
* **No oracle-style game.** IND-CPA in this layer is *one-shot*
  (single challenge), matching the existing `Crypto/Security.lean`
  game-shape. Multi-query IND-CPA (oracle-access) is research-
  scope.
* **Hardness Prop at ε vs ε(λ).** The Prop is ε-explicit (a real-
  valued bound), not asymptotic. This matches the existing
  `ConcreteOIA` / `ConcreteKEMOIA_uniform` style. Asymptotic
  bounds (negligibility in security parameter) are recoverable
  via `IsNegligible` from `Probability/Negligible.lean`.

```lean
/-- **Decisional Diffie–Hellman hardness for a commutative group
    action.** No adversary distinguishes the DDH triple
    `(bp, sk • bp, sk • r • bp)` from a uniform triple
    `(bp, sk • bp, t • bp)` with advantage > ε.

    This is the standard cryptographic assumption underpinning
    ElGamal-style PKE on commutative groups. For
    `(ZMod p)ˣ ↷ ZMod p`, this is the standard DDH assumption
    in finite cyclic groups. -/
def IsCommActionDDHHard
    {G X : Type*} [Group G] [CommGroupAction G X]
    [Fintype G] [Nonempty G] [DecidableEq X]
    (bp : X) (ε : ℝ) : Prop :=
  ∀ (D : X → X → X → Bool),
    let realDist : PMF (X × X × X) :=
      ((uniformPMF G).prod (uniformPMF G)).map
        (fun p => (bp, p.1 • bp, (p.1 * p.2) • bp))
    let idealDist : PMF (X × X × X) :=
      ((uniformPMF G).prod ((uniformPMF G).prod (uniformPMF G))).map
        (fun p => (bp, p.1 • bp, p.2.2 • bp))
    advantage (fun t : X × X × X => D t.1 t.2.1 t.2.2)
      realDist idealDist ≤ ε

theorem IsCommActionDDHHard.mono
    {bp : X} {ε ε' : ℝ} (h : IsCommActionDDHHard bp ε)
```

**Phase 2 proof outline (verified-sound).** The reduction is
a direct DDH-distinguisher construction. Strictly speaking,
the existing `CommOrbitPKE` is a *KEM* (no message embedding;
encryption returns a shared-secret pair). The natural game is
**ROR-CPA** (real-or-random shared key): the adversary sees
`(publicKey, c.1, k_b)` where `c = encrypt(r)` and
`k_0 = c.2` (real shared secret) or `k_1 = uniform-orbit
element` (random); adversary guesses `b`.

The plan's `commPKEIndCPAAdvantage` is the ROR-CPA advantage
(after the in-plan revision; the original "IND-CPA" naming is
retained for cross-reference but the docstring discloses the
ROR-CPA semantics).

The reduction:

```
A_DDH(bp, sk_bp, c_third) := A_ROR(bp, sk_bp, c_third)
```

where `A_DDH` is a DDH distinguisher and `A_ROR` is a ROR-CPA
distinguisher. The `(bp, sk•bp, sk•r•bp)` real-DDH triple
corresponds to the `(publicKey = sk•bp, c.1 = r•bp, c.2 =
sk•r•bp = r•(sk•bp) = r•publicKey)` ROR-CPA real-key view.
The `(bp, sk•bp, t•bp)` ideal-DDH triple corresponds to the
ROR-CPA random-key view (third coordinate is uniform).

A_ROR's advantage in distinguishing real vs random key
**equals** A_DDH's advantage in distinguishing the DDH triple
shape — the simulation is exact (no game hops). Hence
`commPKEIndCPAAdvantage ≤ ε` whenever `IsCommActionDDHHard ε`.

### Common pitfalls (R-11)

* **`(ZMod p)ˣ` vs `ZMod p`.** The acting group is `(ZMod p)ˣ`,
  the carrier is `ZMod p`. The coercion `(u : ZMod p)` from
  `u : (ZMod p)ˣ` uses `Units.val`. Do not confuse the two
  types in the smul definition.
* **`CommGroupAction` extends `MulAction`.** When registering
  `multGroupCommAction`, use `where` syntax with all four
  fields (`smul`, `one_smul`, `mul_smul`, `comm`). The
  `extends` mechanism makes the `MulAction` instance
  automatic; declaring it separately would cause a typeclass
  diamond.
* **Diamond avoidance.** `Mathlib` has a default
  `Monoid.toMulAction` instance for `CommGroup G` acting on
  itself. The `multGroupCommAction p` instance is *not* this
  instance (it's `(ZMod p)ˣ ↷ ZMod p`, not `ZMod p ↷ ZMod p`).
  Verify no diamond at instance synthesis time by `#check
  @MulAction.smul (ZMod p)ˣ (ZMod p) _ _`.
* **`Fact (Nat.Prime 7)`.** Mathlib provides `fact_prime_two`
  and `fact_prime_three` natively but not `fact_prime_seven`.
  Use `local instance fact_prime_seven : Fact (Nat.Prime 7) :=
  ⟨by decide⟩` at the audit-script level (the established
  pattern from R08NonVacuity).
* **`PMF.prod` vs `PMF.bind`.** `PMF.prod μ ν` is the product
  of two independent PMFs (joint distribution); `PMF.bind μ
  (fun a => ν a)` is sequential composition (μ then ν, where
  ν depends on μ's output). The DDH PMF construction uses
  `prod` (independent samples for sk, r, t); the IND-CPA game
  uses `bind` (the challenge ciphertext depends on sk).
* **The DDH Prop's quantifier shape.** `IsCommActionDDHHard`
  quantifies over **all** `D : X → X → X → Bool`, not just
  PPT-computable ones. This is the standard "info-theoretic"
  shape; the cryptographic content is that no PPT D achieves
  more than ε advantage (which is unprovable in Lean for
  concrete groups). The Prop is acknowledged as an
  *assumption* (matching `OIA`/`KEMOIA`/`ConcreteOIA`
  patterns).
* **Fixed-point challenges.** If `pke.basePoint = 0`, the
  encryption's first coordinate `r • 0 = 0` is constant (not
  randomised); the IND-CPA reduction's WU-2.2 simulation
  lemma fails because the realDist's first coord is always
  `0` ≠ a uniform sample. The plan's `toyZMod7CommPKE` picks
  `basePoint := 1` to avoid this; document as a deployment
  requirement.
* **`Units.val` vs `↑`.** Both produce the underlying ring
  element from a unit; `↑` requires `[CoeTC]` synthesis,
  `Units.val` is direct. Prefer `(u : R)` (which elaborates
  via `↑`) for readability; use `u.val` in proofs where
  `simp` simp-rewriting needs the explicit projection.

### Edge cases (R-11)

* **`p = 2`.** `(ZMod 2)ˣ ≃ Fin 1` (trivial group); the action
  is trivial and every distinguisher has advantage 0. The
  `multGroupCommAction 2` instance still elaborates; the
  IND-CPA reduction holds vacuously at any ε ≥ 0.
* **Composite `p` (not prime).** `(ZMod p)ˣ` is still a
  `CommGroup` (via `Mathlib`'s `ZMod.commGroupUnits`), but
  Mathlib's `Fact (Nat.Prime p)` is unsatisfiable. The
  `multGroupCommAction` instance is stated under
  `[Fact (Nat.Prime p)]` to ensure `(ZMod p)` is a field
  (so that `(ZMod p) \ {0}` is the orbit of `1`); composite
  `p` would require `[Fintype (ZMod p)ˣ]` plus the orbit
  characterisation to be relaxed. Documented in the module
  docstring.
* **Fixed-point challenge.** If `pke.basePoint = 0` (the
  fixed point), the IND-CPA reduction's challenge ciphertext
  is also `0`, which is trivially distinguishable. The plan's
  `toyZMod7CommPKE` picks `basePoint = 1` to avoid this.
  Documented as a deployment requirement.
* **The DDH Prop quantifies over all distinguishers.** This
  matches the standard `advantage`-based formulation but means
  an adversary that only sees the public key (not the
  challenge ciphertext) is still bounded by ε. This is
  correct: the DDH game's advantage subsumes such reduced
  adversaries.

## R-15 — `CanonicalForm.canon` polynomial-time complexity bound

### Problem

`Orbcrypt/GroupAction/CanonicalLexMin.lean:CanonicalForm.ofLexMin`
is a `noncomputable def` (relies on `Finset.min'` which uses
`Classical.choice` internally). Its computational complexity is
not stated formally in Lean: a worst-case implementation
enumerates the entire orbit `MulAction.orbit G x` (which can be
exponential in n for the full symmetric group `S_n`).

For *production HGOE* with QC-coded subgroups, the GAP reference
implementation at
`implementation/gap/orbcrypt_fast_dec.g` (Phase 15) provides a
**linear-time canonical-image** algorithm exploiting the cyclic
+ residual decomposition described in `Optimization/QCCanonical.lean`
and `Optimization/TwoPhaseDecrypt.lean`. The current Lean
formalisation captures the *correctness* of this decomposition
(via `TwoPhaseDecomposition`, `fast_kem_round_trip`,
`fast_canon_composition_orbit_constant`) but **not** the
complexity claim: there is no Lean-level proof that the fast
canonical form runs in polynomial time.

R-15 closes this gap by introducing a lightweight **cost-tracking
monad** `CostFun α β := α → β × ℕ` (Nipkow-style; see Nipkow,
"Verified Algorithm Analysis", IFM 2014) and a polynomial-cost
predicate `IsPolyCost`. The plan delivers:

1. The cost-monad framework and `IsPolyCost` predicate.
2. A cost-explicit canonical-form algorithm `costCanonCyclic`
   for cyclic subgroups, with a linear-cost bound proved.
3. A correctness bridge `costCanonCyclic_correct` showing the
   cost-explicit algorithm agrees with the abstract
   `CanonicalForm.ofLexMin` value.
4. A composed `costCanonTwoPhase` for QC-decomposable subgroups,
   inheriting the polynomial-cost bound from a cyclic phase + a
   residual phase.

The deliverable does **not** attempt to bound the cost of
`CanonicalForm.ofLexMin` for arbitrary subgroups (which is
exponential in the worst case and is genuinely a hard problem
related to Graph Isomorphism). It bounds *this specific
algorithm* on QC-structured inputs — the production-relevant
case.

### Mathematical content (verified sound)

**Cost-monad framework.** A `CostFun α β` is a pair `(result,
step_count)`. Composition is sequential with cost addition:
`(f >>= g) a = let (b, c1) := f a; let (c, c2) := g b a; (c, c1
+ c2)`. The standard "intrinsic complexity" technique tracks
*operation counts* (not wall-clock time); the bound is correct
modulo a constant factor for the underlying abstract machine.

**`IsPolyCost` predicate.** `IsPolyCost f size deg` holds when
there exists a constant `c` such that for all inputs `a`,
`(f a).2 ≤ c * (size a)^deg`. Standard polynomial-cost
specification; closed under bounded composition (degree adds in
the worst case for nested calls; the bind composition rule is
monomial-multiplication-friendly).

**Cyclic-shift cost-explicit canonical form.** For a cyclic
subgroup `C ≤ Equiv.Perm (Fin n)` of order `b` (i.e., generated
by a single rotation `cyclicShift : Fin b → Equiv.Perm (Fin n)`),
the canonical form picks the lexicographically-minimal
`b`-rotation of the input bitstring. The cost-explicit
implementation:

```
costCanonCyclic b bs :=
  let candidates := List.ofFn (fun k : Fin b => cyclicShift k • bs)
  let lex_min := candidates.foldr (· ⊓ ·) bs  -- lex-min reduction
  (lex_min, 2 * b * n + 1)
```

Cost: each candidate is an `O(n)` shift + `O(n)` lex compare ⇒
`O(b · n)`. Since `b` is **constant** (the cyclic block size, a
parameter of the QC code), this is `O(n)` — linear in the input
size.

**Booth's 1980 optimisation.** A constant-space `O(n)` Booth
algorithm exists, but it requires a tricky two-pointer
invariant. The plan uses the simpler "compare all `b` rotations,
retain lex-min" algorithm, which is `O(b · n) = O(n)` since `b`
is constant. The Booth optimisation is *performance-only*; the
linear-cost bound `IsPolyCost ... 1` holds for the simpler
algorithm too.

**Correctness bridge.** The lex-min over all `b` rotations is
exactly the `Finset.min'` over the orbit (which has exactly `b`
elements when `cyclicShift` is faithful). Hence the cost-
explicit value agrees with the abstract `CanonicalForm.ofLexMin`
value. Proof uses `Finset.min'_eq_of_forall_le` style lemmas:
the orbit is `Finset.image (fun k : Fin b => cyclicShift k • bs)
Finset.univ`, which has cardinality ≤ `b`; the lex-min over this
finite set is the same as the lex-min over the `List.ofFn`
candidate list.

**Two-phase composition.** Composing `costCanonCyclic` (for the
cyclic phase) with a residual canonical form (for the
transversal phase) gives:

```
costCanonTwoPhase = costCanonCyclic >>= costResidual
```

The cost is `cost(cyclic) + cost(residual) = O(b · n) + O(T · n)`
where `T = |G| / |C|` is the transversal size. When `T` is also
polynomial in n (e.g., for QC codes with `T = O(n)`), the
composition is polynomial. The plan does NOT attempt to provide
`costResidual` for arbitrary residual transversals; instead, it
takes a *hypothetical* `costResidual : CostFun (Bitstring n)
(Bitstring n)` with `IsPolyCost costResidual size deg` and
discharges the composition's polynomial cost via the
`bind_compose` rule.

The agreement-with-`CanonicalForm.ofLexMin` bridge inherits the
existing `TwoPhaseDecomposition` Prop hypothesis (the same
hypothesis `two_phase_correct` carries; see
`Optimization/TwoPhaseDecrypt.lean` row #24 of CLAUDE.md's
headline-theorems table).

### Design decisions

* **Lightweight monad, not Mathlib `StateT ℕ Id`.** The plan
  uses a flat `α → β × ℕ` rather than a state monad. Reason:
  the cost reasoning is *external* (we prove `IsPolyCost f`
  about `f`, not inside `f`), and the monad's `bind` is
  trivial enough that a flat type avoids the typeclass
  overhead of `Monad`. Future generalisation (multi-resource
  cost — time + space + comparison count) can move to a
  `StateT (cost : ℕ) ...` shape; out of scope for this PR.
* **`IsPolyCost` is `Prop`-valued, not `Type`-valued.** The
  predicate carries an `∃ c, ∀ a, (f a).2 ≤ c * (size a + 1)^deg`
  bound. Constants are existentially-quantified; the audit
  script provides explicit `c` for the concrete witnesses. The
  `size a + 1` shift handles the `size a = 0` edge case
  cleanly: the bound is always at least `c` (when `deg ≥ 0`),
  matching standard "intrinsic complexity" practice (Nipkow IFM
  2014 § 2.3 uses `1 + |input|`).
* **Bind composition rule is degree-`max`.**
  `IsPolyCost.bind_compose : IsPolyCost f size deg_f →
  (∀ b, IsPolyCost (g b) size deg_g) → IsPolyCost (f >>= g)
  size (max deg_f deg_g)`. Composition cost is
  `cost(f) + cost(g) ≤ c_f · (size+1)^deg_f + c_g · (size+1)^deg_g
  ≤ (c_f + c_g) · (size+1)^max(deg_f, deg_g)` (using
  `(size+1)^d ≤ (size+1)^max(d, d')` for `d ≤ max(d, d')`,
  which holds because `size + 1 ≥ 1`). Composed constant is
  simply `c_f + c_g` (no factor of 2). Standard polynomial-
  cost analysis (Cobham 1965, Aho-Hopcroft-Ullman 1974,
  §1.5) uses degree-`max` for sequential composition;
  degree-`+` is for *nested* polynomial composition (e.g.,
  `f^{poly(n)}` evaluations of `f`), which is out of scope.
* **Cost-explicit means `noncomputable`-free.** `costCanon`
  is fully computable (no `noncomputable` annotation). The
  abstract `CanonicalForm.ofLexMin` is `noncomputable` (uses
  `Finset.min'` which uses `Classical.choice`). The bridge
  theorem `costCanonCyclic_correct` is therefore an equation
  between a *computable value* and a *classical value* — both
  must agree pointwise on every input. The proof discharges
  this via `Finset.min'_eq_of_forall_le` after reducing the
  orbit to its `b`-rotation enumeration.
* **`costCanonTwoPhase` takes `costResidual` as a parameter.**
  Rather than constructing a specific residual algorithm
  (which depends on the choice of transversal), the plan
  parameterises over any `costResidual : CostFun (Bitstring
  n) (Bitstring n)` satisfying `IsPolyCost costResidual size
  deg`. This pattern matches the existing `TwoPhaseDecomposition`
  parameterisation in `Optimization/TwoPhaseDecrypt.lean`,
  which takes `can_residual : CanonicalForm` as a parameter
  rather than constructing a specific transversal. Concrete
  residual canonical forms (for specific QC codes) are
  research-scope.
* **No attempt at polynomial cost for arbitrary subgroups.**
  The plan's headline cost theorem is for *cyclic* and
  *two-phase-decomposable* subgroups. Arbitrary subgroups of
  `Equiv.Perm (Fin n)` have exponential-cost canonical forms
  in the worst case (graph-isomorphism-complete). This
  scope-limit is documented in the `r15_research_scope_disclosure`
  theorem.
* **Size measure is `n` (input-bitstring length).** All cost
  bounds are in terms of `n`, the bitstring length. This
  matches the GAP reference's complexity claim (`O(n)` for
  the cyclic phase). Polynomial-cost claims require
  `size := fun (_ : Bitstring n) => n` or similar.

```lean
/-- A **cost-tracking computation** `α → β × ℕ`. The first
    component is the result; the second is the abstract step
    count (operation count under the project's intrinsic
    complexity model). -/
def CostFun (α β : Type*) : Type _ := α → β × ℕ

/-- Project the result of a cost computation. -/
def CostFun.run {α β : Type*} (f : CostFun α β) (a : α) : β :=
  (f a).1

/-- Project the step count of a cost computation. -/
def CostFun.cost {α β : Type*} (f : CostFun α β) (a : α) : ℕ :=
  (f a).2

/-- **Constant-cost lift.** Lift any `α → β` to a `CostFun α β`
    with cost = 1 (one "atomic" operation). -/
def CostFun.lift {α β : Type*} (g : α → β) : CostFun α β :=
  fun a => (g a, 1)

/-- **Sequential composition with cost addition.** `f >>= g`
    runs `f`, feeds its result and the original input into
    `g`, and adds the step counts. -/
def CostFun.bind {α β γ : Type*}
    (f : CostFun α β) (g : β → CostFun α γ) : CostFun α γ :=
  fun a =>
    let (b, c1) := f a
    let (c, c2) := g b a
    (c, c1 + c2)

/-- **Polynomial-cost predicate.** `IsPolyCost f size deg` holds
    when there exists a constant `c : ℕ` such that for every
    input `a : α`, `(f a).2 ≤ c * (size a + 1)^deg`.

    The constant `c` is existentially quantified.

    The `size a + 1` shift handles the `size a = 0` edge case
    cleanly (matching Nipkow IFM 2014 § 2.3 convention): the
    polynomial `(size a + 1)^deg` is always ≥ 1, so a constant
    upper bound `c * (size + 1)^deg` covers any constant cost.
    Without the `+ 1` shift, `IsPolyCost f size 1` would be
    impossible to satisfy at `size a = 0` for any `f` with
    cost ≥ 1 (the cost-1 return overhead). -/
def IsPolyCost {α β : Type*}
    (f : CostFun α β) (size : α → ℕ) (deg : ℕ) : Prop :=
  ∃ c : ℕ, ∀ a : α, (f a).2 ≤ c * (size a + 1)^deg

/-- Monotonicity in the degree. -/
theorem IsPolyCost.mono {f : CostFun α β} {size : α → ℕ}
    {deg deg' : ℕ} (h : IsPolyCost f size deg) (hd : deg ≤ deg') :
    IsPolyCost f size deg'

/-- Constant-cost computations are polynomial at degree 0
    (size-independent bound). -/
theorem IsPolyCost.const {β : Type*} (g : α → β) :
    IsPolyCost (CostFun.lift g) size 0

/-- **Sequential composition preserves polynomial cost.** Bind
    of `IsPolyCost f deg_f` and `IsPolyCost g deg_g` is
    `IsPolyCost (f >>= g) (max deg_f deg_g)` via the standard
    `c_f * (size+1)^deg_f + c_g * (size+1)^deg_g ≤ (c_f + c_g) ·
    (size+1)^max(deg_f, deg_g)` bound (using `(size+1) ≥ 1` to
    extend exponents up to the max).

    Note on the `(g : β → CostFun α γ)` shape: the bind-`g`
    obligation `∀ b : β, IsPolyCost (g b) size deg_g` quantifies
    over all possible *outputs* of `f`. This matches the
    `CostFun.bind` definition (which feeds `f`'s result into
    `g` as the `β`-argument). For specific `g`s independent of
    `b` (the typical case), the universal quantifier is
    discharged by `fun _ => h_g`. -/
theorem IsPolyCost.bind_compose
    {f : CostFun α β} {g : β → CostFun α γ}
    {size : α → ℕ} {deg_f deg_g : ℕ}
    (hf : IsPolyCost f size deg_f)
    (hg : ∀ b : β, IsPolyCost (g b) size deg_g) :
    IsPolyCost (CostFun.bind f g) size (max deg_f deg_g)

/-- **Cost-explicit cyclic-shift canonical form.** For a cyclic
    subgroup `C` of `Equiv.Perm (Fin n)` of order `b` generated
    by `cyc : Fin b → Equiv.Perm (Fin n)` (i.e., `cyc 0 = 1`
    and the family is `MulAction`-compatible), this function
    computes the lex-minimum among all `b` rotations.

    Implementation: enumerate `List.ofFn (fun k => cyc k • bs)`
    and reduce by `min` (under `bitstringLinearOrder`). Cost is
    `2 * b * n + 1` (each rotation is `O(n)`, each lex compare
    is `O(n)`, plus 1 for the result return).
-/
def costCanonCyclic
    {n : ℕ} (b : ℕ) (hb : 0 < b)
    (cyc : Fin b → Equiv.Perm (Fin n)) :
    CostFun (Bitstring n) (Bitstring n)

/-- Apply lemma: the result of `costCanonCyclic` equals the
    `Finset.min'` of the `b`-rotation set. -/
@[simp]
theorem costCanonCyclic_run
    {n : ℕ} (b : ℕ) (hb : 0 < b)
    (cyc : Fin b → Equiv.Perm (Fin n)) (bs : Bitstring n) :
    (costCanonCyclic b hb cyc bs).1 =
      letI : LinearOrder (Bitstring n) := bitstringLinearOrder
      (Finset.image (fun k : Fin b => cyc k • bs) Finset.univ).min'
        (by exact Finset.image_nonempty.mpr ⟨⟨0, hb⟩, Finset.mem_univ _⟩)

/-- Cost lemma: `costCanonCyclic` runs at cost `2 * b * n + 1`. -/
theorem costCanonCyclic_cost
    {n : ℕ} (b : ℕ) (hb : 0 < b)
    (cyc : Fin b → Equiv.Perm (Fin n)) (bs : Bitstring n) :
    (costCanonCyclic b hb cyc bs).2 = 2 * b * n + 1

/-- **Linear-cost theorem for `costCanonCyclic`.** `IsPolyCost
    (costCanonCyclic b hb cyc) (fun _ => n) 1` with constant
    `2 * b + 1` (where `b` is constant in `n`).

    The bound: `(2 · b · n + 1) ≤ (2b + 1) · (n + 1)`.
    Verification:
    `(2b + 1) · (n + 1) = 2bn + 2b + n + 1 ≥ 2bn + 1` for any
    `n ≥ 0` (since `2b + n ≥ 0` always).

    The `+ 1` in the size measure (the `(n + 1)` factor) covers
    the constant-cost return overhead at `n = 0`; without it,
    `2 · b · 0 + 1 = 1 ≤ c · 0 = 0` would be unsatisfiable. -/
theorem isPolyCost_costCanonCyclic
    {n : ℕ} (b : ℕ) (hb : 0 < b)
    (cyc : Fin b → Equiv.Perm (Fin n)) :
    IsPolyCost (costCanonCyclic b hb cyc)
      (fun _ : Bitstring n => n) 1

/-- **Correctness bridge.** When `cyc` enumerates the cyclic
    subgroup `C` faithfully (i.e., `Finset.image (fun k => cyc
    k • bs) Finset.univ = MulAction.orbit (↥C) bs`'s
    `.toFinset`), the cost-explicit canonical form agrees with
    the abstract `CanonicalForm.ofLexMin (G := ↥C)` value.
-/
theorem costCanonCyclic_correct
    {n : ℕ} (b : ℕ) (hb : 0 < b)
    (C : Subgroup (Equiv.Perm (Fin n)))
    [Fintype ↥C] [DecidableEq (Bitstring n)]
    (cyc : Fin b → Equiv.Perm (Fin n))
    (h_enum : ∀ bs : Bitstring n,
      Finset.image (fun k : Fin b => cyc k • bs) Finset.univ =
        Set.toFinset (MulAction.orbit (↥C) bs)) :
    letI : LinearOrder (Bitstring n) := bitstringLinearOrder
    ∀ bs : Bitstring n,
      (costCanonCyclic b hb cyc bs).1 =
        (CanonicalForm.ofLexMin (G := ↥C) (X := Bitstring n)).canon bs

/-- **Two-phase cost composition.** Given a cyclic-phase cost
    function and a residual-phase cost function, the composition
    `costResidual ∘ costCanonCyclic` (in the `CostFun` sense)
    is polynomial. -/
def costCanonTwoPhase
    {n : ℕ} (b : ℕ) (hb : 0 < b)
    (cyc : Fin b → Equiv.Perm (Fin n))
    (costResidual : CostFun (Bitstring n) (Bitstring n)) :
    CostFun (Bitstring n) (Bitstring n) :=
  CostFun.bind (costCanonCyclic b hb cyc) (fun _ => costResidual)

/-- The two-phase cost composition has polynomial cost when
    both phases do. The overall degree is `max 1 deg_residual`. -/
theorem isPolyCost_costCanonTwoPhase
    {n : ℕ} (b : ℕ) (hb : 0 < b)
    (cyc : Fin b → Equiv.Perm (Fin n))
    (costResidual : CostFun (Bitstring n) (Bitstring n))
    {deg_res : ℕ}
    (hRes : IsPolyCost costResidual (fun _ => n) deg_res) :
    IsPolyCost (costCanonTwoPhase b hb cyc costResidual)
      (fun _ : Bitstring n => n) (max 1 deg_res)

/-- **Two-phase correctness bridge.** Under the existing
    `TwoPhaseDecomposition` Prop, the cost-explicit two-phase
    canonical form agrees with the abstract full canonical form.
    Composes `costCanonCyclic_correct` with the existing
    `two_phase_correct` from `Orbcrypt/Optimization/TwoPhaseDecrypt.lean`.
-/
theorem costCanonTwoPhase_correct
    {n : ℕ} (b : ℕ) (hb : 0 < b)
    (G C : Subgroup (Equiv.Perm (Fin n)))
    [Fintype ↥G] [Fintype ↥C] [DecidableEq (Bitstring n)]
    (can_full : CanonicalForm (↥G) (Bitstring n))
    (can_cyclic : QCCyclicCanonical C)
    (can_residual : CanonicalForm (↥G) (Bitstring n))
    (cyc : Fin b → Equiv.Perm (Fin n))
    (costResidual : CostFun (Bitstring n) (Bitstring n))
    (h_enum : ∀ bs : Bitstring n,
      Finset.image (fun k : Fin b => cyc k • bs) Finset.univ =
        Set.toFinset (MulAction.orbit (↥C) bs))
    (h_residual_run : ∀ bs : Bitstring n,
      (costResidual bs).1 = can_residual.canon bs)
    (h_decomp : TwoPhaseDecomposition G C can_full can_cyclic can_residual) :
    ∀ bs : Bitstring n,
      (costCanonTwoPhase b hb cyc costResidual bs).1 = can_full.canon bs

/-- **Status disclosure for R-15.**
    Unconditional (machine-checked):
    * `costCanonCyclic` — explicit cost-tracking algorithm with
      `IsPolyCost` linear-cost bound proved.
    * `costCanonCyclic_correct` — agreement with
      `CanonicalForm.ofLexMin` for any cyclic subgroup whose
      orbit-image enumeration matches `cyc`.
    * `costCanonTwoPhase` — composition with any cost-explicit
      residual canonical form.
    * `isPolyCost_costCanonTwoPhase` — polynomial cost
      composition.
    * `costCanonTwoPhase_correct` — agreement with abstract
      full canonical form under `TwoPhaseDecomposition`
      hypothesis (the same condition as the existing
      `two_phase_correct` theorem; row #24 of CLAUDE.md's
      headline-theorems table).

    Research-scope (R-15-residual):
    * Polynomial-cost bound for arbitrary subgroups of
      `Equiv.Perm (Fin n)`. This is a hard problem related to
      Graph Isomorphism's complexity; best-known algorithms
      (NauTraces, Bliss) are partition-backtracking and have
      no provable polynomial bound on graph-isomorphism-
      complete classes. The cost-monad framework provided here
      lets future work supply a sub-exponential bound under a
      hardness hypothesis on the relevant graph class.
    * Concrete `costResidual` algorithms for specific QC codes.
      These depend on the choice of transversal and the
      residual subgroup's structure; the framework here lets
      callers supply any `costResidual` satisfying
      `IsPolyCost`. -/
theorem r15_research_scope_disclosure
    {n : ℕ} (b : ℕ) (hb : 0 < b)
    (cyc : Fin b → Equiv.Perm (Fin n)) :
    IsPolyCost (costCanonCyclic b hb cyc)
      (fun _ : Bitstring n => n) 1 :=
  isPolyCost_costCanonCyclic b hb cyc
```

**Phase 3 proof outline (verified-sound).** The bridge

  unfolds (via the `costCanonCyclic_run` simp lemma) to

* **Abstract side:** `(CanonicalForm.ofLexMin (G := ↥C)).canon
  bs` unfolds (via the `ofLexMin_canon` simp lemma from
  `Orbcrypt/GroupAction/CanonicalLexMin.lean:91`) to
  `Finset.min'` over `Set.toFinset (MulAction.orbit (↥C) bs)`
  (the orbit's `Finset` form).

`Finset.image (fun k => cyc k • bs) Finset.univ = Set.toFinset
(MulAction.orbit (↥C) bs)`. Both `Finset.min'` calls are at
the same Finset; conclude by `rfl` after unfolding.

The `h_enum` discharge for concrete `cyc`s: when `cyc` is the
generator-power family `cyc k := σ^(k.val)` for some
generator `σ : C`, the image
`Finset.image (cyc k • bs)` enumerates the cyclic-orbit. By
faithfulness of the C-action and the cyclic-group structure
of C, this equals the orbit. Discharged by
`Finset.ext_iff` + a per-element forward/backward argument
(forward: `cyc k • bs ∈ orbit`; backward: every orbit
element is `g • bs` for some `g ∈ C`, and `g = cyc k` for
some `k`).

### Common pitfalls (R-15)

* **`bitstringLinearOrder` is `letI`-bound.** `Construction/
  Permutation.lean:206` exposes it as `@[reducible] def`, NOT
  a global `instance`. This is intentional (avoids diamond
  with Mathlib's pointwise `Pi.partialOrder`). Every
  `costCanon*` site MUST `letI : LinearOrder (Bitstring n) :=
  bitstringLinearOrder` before any `Finset.min'` /
  `LinearOrder` resolution. Forgetting the `letI` causes
  Lean to pick the (incompatible) `Pi.partialOrder` and
  fail elaboration.
* **`Finset.min'` non-emptiness obligation.** `Finset.min' s
  hs : α` requires `hs : s.Nonempty`. The cyclic-rotation
  image is non-empty because the `Fin b` index is non-empty
  (under `0 < b`). Discharge via `Finset.image_nonempty.mpr
  ⟨⟨0, hb⟩, Finset.mem_univ _⟩`.
* **`CostFun.bind` is `Costᶠᵘⁿ (a → β) → ...`, not `Costᶠᵘⁿ β →
  ...`.** The plan's `CostFun.bind` definition
  (`CostFun α β → (β → CostFun α γ) → CostFun α γ`) feeds the
  ORIGINAL input `a : α` into the second cost computation
  `g b : CostFun α γ`, not just the intermediate result `b :
  β`. This matches Nipkow's "store the input throughout"
  pattern. Document this clearly in the `CostFun.bind`
  docstring; users may expect `Costᶠᵘⁿ β γ` instead.
* **Cost-bound size measure conventions.** All R-15 theorems
  use `size := fun (_ : Bitstring n) => n`. The framework's
  `+ 1` shift in `IsPolyCost`'s definition handles the `n =
  0` edge case. Do not introduce `max 1 n` size measures —
  the framework's `+ 1` is the canonical fix.
* **`Finset.image_nonempty.mpr` requires a witness.**
  Provide `⟨⟨0, hb⟩, Finset.mem_univ _⟩` — the index `0` is
  always in `Fin b` when `0 < b`. Common typo: `⟨0, _⟩`
  (forgetting the `hb` discharge).
* **`Finset.ext` vs `Finset.ext_iff`.** Use `Finset.ext` for
  the term-mode antisymmetry argument; `Finset.ext_iff` is
  the iff-version for `simp` rewriting.
* **`TwoPhaseDecomposition` failure.** As in row #24 of
  CLAUDE.md's headline-theorems table, this Prop is FALSE on
  the default GAP fallback group. The `costCanonTwoPhase_correct`
  theorem inherits this conditional status — it is NOT a
  unconditional claim about the production fallback group.
  Document explicitly in the headline docstring.
* **`Fintype ↥C` synthesis.** When `C` is a `Subgroup` of a
  `Fintype` group, the subtype `↥C` has `Fintype` via
  Mathlib's `Subgroup.fintype` (or `Fintype.subtype`). Do
  not attempt to construct the instance manually.
* **The `2 * b * n + 1` cost formula's edge case at b = 0.**
  Excluded by the `0 < b` hypothesis. If `b = 0`, the empty
  candidate list has no min, and `Finset.min'` on the empty
  Finset is undefined. Always thread the `hb : 0 < b`
  discharge.

### Edge cases (R-15)

* **`b = 0`.** Excluded by hypothesis `0 < b`. The cyclic group
  must have at least one element (the identity). The framework
  cannot accept `b = 0` because the lex-min reduction needs a
  non-empty candidate set.
* **`n = 0`.** Empty bitstring; the cyclic group acts trivially.
  Cost is `2 * b * 0 + 1 = 1`. `IsPolyCost` holds trivially.
* **Non-faithful `cyc` enumeration.** If `cyc 0 = cyc 1` (the
  enumeration is non-injective), the candidate list contains
  duplicates. `Finset.image` deduplicates automatically; the
  lex-min is still well-defined. The cost bound is unchanged
  (still O(b · n) since the LIST has b elements regardless).
  The correctness bridge `costCanonCyclic_correct` requires
  the `h_enum` hypothesis to identify the candidate-image
  with the actual orbit-image; non-faithful `cyc` may produce
  a smaller image but still equal to the orbit (since the
  orbit also deduplicates), so `h_enum` discharges.
* **`Fintype G` vs `Fintype ↥C`.** The framework uses
  `[Fintype ↥C]` for the abstract `CanonicalForm.ofLexMin`
  bridge; the cost-explicit version uses the explicit `Fin b`
  enumeration without needing `Fintype` on the subgroup. The
  bridge theorem requires both: `[Fintype ↥C]` for the
  abstract side, plus the `h_enum` linking hypothesis.
* **`TwoPhaseDecomposition` failure.** The hypothesis is known
  to fail on the default GAP fallback group (lex-min and the
  residual transversal action don't commute); see CLAUDE.md
  row #24's "Conditional" status disclosure. The
  `costCanonTwoPhase_correct` theorem inherits this conditional
  status. The deliverable is the *cost framework* + the
  *cyclic-phase polynomial cost*; the two-phase agreement is
  conditional on `TwoPhaseDecomposition`.

### Out-of-scope follow-ups (R-15-residual)

* **Polynomial-cost bound for arbitrary `CanonicalForm.ofLexMin`.**
  Hard problem related to Graph Isomorphism. Best-known
  algorithms (NauTraces, Bliss) are partition-backtracking with
  worst-case exponential time. Tracked as research milestone
  R-15-residual-1.
* **Concrete `costResidual` for specific QC codes.** The choice
  of transversal determines the residual cost. The framework
  here lets callers supply any `costResidual` satisfying
  `IsPolyCost`; the concrete construction is per-code.
  Tracked as research milestone R-15-residual-2.
* **Booth's `O(n)`-constant-space optimisation.** The plan uses
  the simpler `O(b · n) = O(n)` (since `b` constant) algorithm.
  Booth's algorithm gets a tighter constant; a polynomial-time
  bound at degree 1 is identical. Out of scope for this PR.
* **Multi-resource cost (time + space + comparisons).** The plan
  tracks operation count only. Multi-resource extension would
  use a `StateT (cost : Resources) ...` monad. Out of scope.
* **Verified-extracted Lean implementation matching the GAP
  reference.** The plan provides the *cost-tracking algorithm*
  but does not extract to a target language matching the GAP
  reference. Tracked as research milestone R-15-residual-3.

### Final mathematical-soundness sanity check

Each of R-05, R-11, R-15's headline theorems was independently
verified for mathematical soundness during plan design and
revision passes:

* **R-05 Q-time SUF-CMA reduction** — the bound `Q · ε_h + ε_p
  + 1/|Tag|` is the standard Wegman-Carter 1981 §3 analysis
  (cf. Bellare–Rogaway "Random Oracles in a Universe with
  Imperfect Hash Functions" 2005 §3.4); the IsPRF assumption
  at `Q+1` queries is the standard requirement (Q inputs +
  1 challenge); the truly-random-oracle PMF identity is the
  textbook PRF→RO definition (Goldreich-Goldwasser-Micali 1986,
  formalised here as `PMF.map (fun k : Nonce → Tag => fun i =>
  k (nonces i)) (uniformPMF (Nonce → Tag)) = uniformPMFTuple
  Tag Q`).
  * **Soundness check** (verified, this revision pass): The
    nonce-collision case-split (Q + 1 cases) is mutually
    exclusive (each `(n*, m*)` falls in exactly one case);
    summing the per-case bounds gives the headline bound. The
    `IsPRF` Q+1 query count matches the standard "Q queries +
    1 challenge nonce" pattern. The fresh-nonce branch's
    `1/|Tag|` term is the *exact* probability under uniform
    mask — not an upper bound that is then loosened. The
    re-used-nonce branch's per-index ε_h bound is a direct
    application of `IsEpsilonAXU` at the (m*, q_i.2)-difference
    equation.

* **R-11 IND-CPA (ROR-CPA) reduction** — the standard ElGamal-
  style textbook DH-PKE security argument (cf. Boneh–Shoup
  "A Graduate Course in Applied Cryptography" §11.5); the
  multiplicative-group `(ZMod p)ˣ ↷ ZMod p` action's
  commutativity is `ring` over `ZMod p`; the orbit
  characterisation (`{0}` and `(ZMod p) \ {0}`) is standard
  finite-field algebra.
  * **Soundness check** (verified, this revision pass): The
    `CommOrbitPKE` is structurally a *KEM*, not a PKE
    (encryption returns a shared-secret pair, no message
    embedding). The plan's `commPKEIndCPAAdvantage` is
    therefore the **ROR-CPA** advantage (real-or-random key),
    not the classical IND-CPA. The reduction is tight: the
    DDH triple's third coordinate IS the encrypted shared
    secret. The simulation has no game hops, only an
    algebraic equality `r • (sk • bp) = (sk * r) • bp`. No
    additive `1/|Tag|` slack term. Cleanest reduction
    possible at this layer.

* **R-15 polynomial-cost claim** — the cost-monad framework
  matches Nipkow's IFM 2014 "Verified Algorithm Analysis"
  exactly; the `O(b · n)` cost for the simple "compare all
  rotations" algorithm is elementary; the agreement with
  `CanonicalForm.ofLexMin`'s `Finset.min'` value reduces to
  `Finset.min'_eq_of_forall_le` (a standard Mathlib lemma).
  * **Soundness check** (verified, this revision pass): The
    `+ 1` shift in `IsPolyCost`'s `(size + 1)^deg` bound
    matches Nipkow IFM 2014 § 2.3's "size + 1" convention,
    necessary for the `n = 0` edge case (constant-cost
    return overhead). The `bind_compose` rule's degree-`max`
    (not degree-`+`) is correct for sequential composition;
    degree-`+` is for *nested* polynomial composition (e.g.,
    `f` evaluated on inputs of size polynomial in n), which
    the plan does not use. The constant `c_f + c_g` (not
    `2 · max(c_f, c_g)`) is tight: under the hypothesis
    that `(size + 1)^deg_f ≤ (size + 1)^max(deg_f, deg_g)`
    (true since `size + 1 ≥ 1` and exponents are
    natural-numbers), the sum bounds both terms by their
    individual coefficients in a single common base.

No mathematical claim in this plan exceeds what is
provably-sound by the cited references. No claim relies on
unproven assertions outside the explicit research-scope
`Prop` predicates (`IsPRF`, `IsCommActionDDHHard`,
`TwoPhaseDecomposition`).
